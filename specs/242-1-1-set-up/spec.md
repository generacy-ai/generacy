### Release Streams
All repos use `develop` as default branch, `main` for stable releases. This setup must support:
- **Preview stream**: On merge to `develop`, publish snapshot versions to npm with `@preview` dist-tag (e.g., `1.0.0-preview.20260224`)
- **Stable stream**: On merge to `main`, run changesets release and publish with `@latest` dist-tag
- Configure changesets for both streams
- Set up branch protection on `main` across all public repos (require PR, require CI)
- Establish `main` branches with initial merge from `develop`

Branch state: `main` exists in all repos but is 30-180 commits behind `develop`.
