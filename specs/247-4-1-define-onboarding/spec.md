## Summary

## 4.1 — Define onboarding PR template content

Define the files that the onboarding PR will add/modify in a developer's primary repo.

### Tasks
- Define template file structure:
  ```
  .generacy/
  ├── config.yaml              # Project ID, repo list, settings
  └── generacy.env.template    # Template for local secrets

  .devcontainer/
  ├── devcontainer.json         # With Generacy Dev Container Feature
  └── docker-compose.yml        # If multi-repo (orchestrator + workers)

  .vscode/
  └── extensions.json           # Recommends Agency + Generacy extensions
  ```
- Template system supporting variable substitution (project ID, repo URLs, etc.)
- Single-repo vs. multi-repo templates
- Language-agnostic base (works with any stack)

### Acceptance Criteria
Templates produce valid devcontainer and Generacy configs for various project shapes.
