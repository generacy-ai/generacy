# Technical Research: Bootstrap CLAUDE.md, .speckit templates, .mcp.json

## Technology Decisions

### 1. CLAUDE.md Format

**Decision**: Follow the claude-plugins CLAUDE.md format

**Rationale**:
- Consistent with existing claude-plugins repository
- Auto-generated format supports update_agent MCP tool
- Sections align with speckit workflow requirements
- Manual additions section allows customization

**Alternatives Considered**:
| Alternative | Pros | Cons |
|------------|------|------|
| Custom format | Tailored to generacy | Incompatible with update_agent tool |
| Minimal README only | Simpler | Missing AI-assistance context |
| Per-component docs | Granular | Harder to maintain consistency |

### 2. MCP Server Selection

**Decision**: Context7 + Playwright (matching agency repo)

**Rationale**:
- Context7: Industry-standard for documentation lookup
- Playwright: Essential for web console testing (future)
- Both are npx-compatible (no local installation needed)
- Proven configuration from agency repository

**Alternatives Considered**:
| Server | Status | Reason |
|--------|--------|--------|
| Redis MCP | Deferred | No Redis integration yet |
| Docker MCP | Deferred | Container support pending |
| Memory MCP | Evaluated | Context7 sufficient for now |

### 3. Template Source

**Decision**: Copy from claude-plugins repository

**Rationale**:
- Templates are actively maintained
- Ensures compatibility with speckit commands
- Avoids divergent template versions
- 5 templates provide complete workflow coverage

**Source Location**: `/workspaces/claude-plugins/.specify/templates/`
