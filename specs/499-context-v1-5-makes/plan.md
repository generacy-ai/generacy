# Implementation Plan: Credential Audit Log

**Feature**: Audit log writer in credhelper-daemon (memory-buffered, ring-bounded)
**Branch**: `499-context-v1-5-makes`
**Status**: Complete

## Summary

Add structured audit logging to `credhelper-daemon` for security investigation of credential operations. A new `audit/` module implements a bounded ring buffer (capacity 5000) that records every credential lifecycle event (mint, resolve, render, session begin/end, proxy allow/deny). Audit batches are flushed to the control-plane via `POST /control-plane/internal/audit-batch`, which emits them on the relay's `cluster.audit` event channel for cloud ingestion. The system never includes credential values, enforced by a dev-mode assertion on field length.

**Language/Version**: TypeScript (ESM), Node >= 20
**Primary Dependencies**: `node:http` (Unix socket HTTP client), `zod` (schema validation), `node:crypto` (existing in daemon)

**Testing**: `vitest` (existing test framework across packages)
**Target Platform**: Linux (in-cluster orchestrator container)
**Project Type**: Monorepo packages (`credhelper-daemon`, `credhelper`, `control-plane`)
**Performance Goals**: Ring buffer O(1) push/drop; batch flush <= 50 entries or 1s interval
**Constraints**: Never OOM from audit pressure; no credential values in entries; field max 256 chars
**Scale/Scope**: 3 packages modified, ~10 new files, ~600-800 lines new code

## Key Design Decisions

1. **Ring buffer over queue**: Fixed-capacity circular buffer with O(1) push. No allocations beyond initial capacity. Oldest entries silently dropped when full — matches "fail-open for availability, fail-closed for secrets" philosophy.

2. **Control-plane as relay bridge**: Daemon pushes batches via HTTP POST to control-plane (which has relay access), rather than giving daemon its own WebSocket. Reuses existing Unix socket infrastructure.

3. **Plugin audit via callback injection**: Rather than modifying every plugin's mint/resolve signature, the SessionManager wraps plugin calls with try/catch and records audit entries at the call site. Plugins remain unaware of audit.

4. **Sampling at hook site**: Docker/localhost proxy hooks apply sampling (1/100 default) at the call site using a deterministic counter, not random. `RoleConfig.audit.recordAllProxy` overrides to 100%.

5. **Batch transport**: Timer-based (1s) with early flush at 50 entries. Single HTTP POST per batch. If control-plane is unavailable, entries stay in ring buffer (and eventually get dropped by capacity limit). No retry — matches bounded-memory guarantee.
