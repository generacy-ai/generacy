## Nullability convention

`viewerDidAuthor` is serialized as literal `null` when the source `comment.viewerDidAuthor` is `undefined`, so downstream log-based dashboards can distinguish "absent" (`null`) from "present-and-false" (`false`) from "present-and-true" (`true`).

Rationale: pino's default serialization drops `undefined` properties, which would make an absent value indistinguishable from a schema regression that never emitted the key. Explicit `?? null` preserves the observability signal FR-002 requires.
