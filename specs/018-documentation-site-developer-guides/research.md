# Research: Documentation site and developer guides

## Technology Decisions

### Docusaurus 3.x (Documentation Framework)

**Decision**: Use Docusaurus 3.x as the primary documentation framework.

**Rationale**:
- React-based ecosystem aligns with broader JavaScript/TypeScript tooling
- Built-in versioning support crucial for multi-component documentation (Agency, Humancy, Generacy)
- Excellent TypeDoc integration via `docusaurus-plugin-typedoc`
- Strong MDX support for interactive documentation components
- Large community and extensive plugin ecosystem
- Good SEO defaults out of the box

**Alternatives Considered**:
- **VitePress**: Lighter, Vue-based, but smaller ecosystem and less mature versioning
- **GitBook**: Commercial, less customizable
- **MkDocs**: Python-based, would add different toolchain

### GitHub Pages (Hosting)

**Decision**: Host on GitHub Pages.

**Rationale**:
- Free for public repositories
- Native GitHub integration (automatic deployment from Actions)
- Custom domain support
- No additional vendor accounts required
- Sufficient for documentation traffic

**Alternatives Considered**:
- **Vercel**: More features but adds complexity and potential costs at scale
- **Netlify**: Similar to Vercel, overkill for static docs
- **CloudFlare Pages**: Good but adds another vendor

### TypeDoc + OpenAPI (API Documentation)

**Decision**: Use TypeDoc for npm packages, OpenAPI for REST endpoints.

**Rationale**:
- TypeDoc generates from TypeScript source - single source of truth for npm packages
- OpenAPI is the industry standard for REST API documentation
- Both integrate well with Docusaurus
- Enables auto-generation from code, reducing doc drift

**Implementation**:
- `docusaurus-plugin-typedoc` for TypeDoc integration
- `@docusaurus/plugin-content-docs` with OpenAPI spec rendering

### Mermaid + Excalidraw (Diagrams)

**Decision**: Use both tools for different diagram types.

**Rationale**:
- **Mermaid**: Version-controllable, renders inline in markdown, ideal for:
  - Sequence diagrams
  - Component interaction flows
  - Data flow diagrams
  - Architecture diagrams
- **Excalidraw**: Hand-drawn aesthetic, ideal for:
  - Conceptual overviews (The Triad)
  - Adoption path visualizations
  - Marketing/brand-aligned visuals

**Implementation**:
- `@docusaurus/theme-mermaid` for inline Mermaid rendering
- Excalidraw files in `static/img/diagrams/` exported as PNG/SVG

## Implementation Patterns

### Progressive Documentation Structure

The documentation follows the progressive adoption model:

```
Level 1: Agency Only → Simple local setup
Level 2: Agency + Humancy → Human oversight added
Level 3: Local Orchestration → Full local stack (future)
Level 4: Cloud Deployment → Enterprise setup (future)
```

This matches how users adopt Generacy and provides natural learning progression.

### MDX Component Patterns

Custom React components for consistent documentation patterns:

```tsx
// AdoptionLevel component for level badges
<AdoptionLevel level={1} title="Agency Only" />

// Code tabs for multi-language examples
<Tabs>
  <TabItem value="npm">npm install @generacy-ai/agency</TabItem>
  <TabItem value="yarn">yarn add @generacy-ai/agency</TabItem>
</Tabs>
```

### API Documentation Generation

**TypeDoc Configuration**:
```json
{
  "entryPoints": ["packages/*/src/index.ts"],
  "out": "docs/api",
  "plugin": ["typedoc-plugin-markdown"]
}
```

**OpenAPI Integration**:
- OpenAPI spec in `static/api/orchestrator.yaml`
- Rendered via Docusaurus OpenAPI plugin or embedded Swagger UI
