# Feature Specification: Define speckit as Generacy workflow actions

**Branch**: `156-define-speckit-generacy-workflow` | **Date**: 2026-02-04 | **Status**: Draft

## Summary

## Description

Create Generacy workflow actions that implement the speckit specification-driven development methodology, enabling workflows to use specify, clarify, plan, tasks, and implement steps.

## Problem

The speckit methodology (specify → clarify → plan → tasks → implement) is currently implemented as MCP tools in claude-plugins. To use this workflow in Generacy, we need to define these as Generacy workflow actions that can be referenced in .generacy.yaml workflow definitions.

## Current Speckit MCP Tools (to migrate)

| Tool | Purpose |
|------|---------|
| `git_ops` | Git operations (branch, checkout, status, fetch) |
| `create_feature` | Create feature branch and spec directory |
| `get_paths` | Get feature specification paths |
| `check_prereqs` | Validate prerequisites |
| `copy_template` | Copy spec/plan/tasks templates |
| `update_agent` | Update AI agent context files |
| `manage_clarifications` | Read/append/update clarifications |
| `tasks_to_issues` | Convert tasks.md to GitHub issues |

## Proposed Solution

### Workflow Actions

Create Generacy actions under the `speckit.*` namespace:

```yaml
# Example workflow using speckit actions
name: feature-development
steps:
  - id: setup
    action: speckit.create_feature
    with:
      description: ${{ inputs.description }}
      
  - id: specify
    action: speckit.specify
    with:
      feature_dir: ${{ steps.setup.outputs.feature_dir }}
      
  - id: clarify
    action: speckit.clarify
    with:
      feature_dir: ${{ steps.setup.outputs.feature_dir }}
    gate: clarification-review
    
  - id: plan
    action: speckit.plan
    with:
      feature_dir: ${{ steps.setup.outputs.feature_dir }}
    gate: plan-review
    
  - id: tasks
    action: speckit.tasks
    with:
      feature_dir: ${{ steps.setup.outputs.feature_dir }}
    gate: tasks-review
    
  - id: implement
    action: speckit.implement
    with:
      feature_dir: ${{ steps.setup.outputs.feature_dir }}
```

### Implementation Strategy

**Option A: Wrap existing MCP tools**
- Create thin action wrappers that call the existing speckit MCP server
- Fastest path to migration
- Maintains compatibility with current implementation

**Option B: Port logic directly**
- Copy/adapt speckit logic into Generacy action implementations
- Cleaner long-term architecture
- More maintenance burden

**Recommended: Option A first**, then refactor to Option B as needed.

### Standard Workflow Templates

Provide pre-built workflow templates:

1. `speckit-feature.yaml` - Standard feature development
2. `speckit-epic.yaml` - Epic with child issue creation
3. `speckit-bugfix.yaml` - Simplified bug fix workflow

## Use Case

As a workflow author, I want to use speckit methodology steps in my Generacy workflows so that I can leverage specification-driven development in automated pipelines.

As an agent, I want standardized workflow templates so that I can follow consistent development practices across projects.

## Acceptance Criteria

1. `speckit.create_feature` action creates branch and spec directory
2. `speckit.specify` action generates spec.md from description/issue
3. `speckit.clarify` action identifies questions and posts to GitHub
4. `speckit.plan` action generates plan.md with implementation approach
5. `speckit.tasks` action generates tasks.md with ordered task list
6. `speckit.implement` action executes tasks with progress tracking
7. All actions support gate/wait for human review
8. Standard workflow templates are available
9. Actions work both in headless mode and VS Code debugging
10. Outputs from each action available to subsequent steps
