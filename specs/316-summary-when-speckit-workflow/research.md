# Research: Post Clarification Questions on Issue

## Technology Decisions

### 1. GitHub API Client vs `gh` CLI

**Decision**: Use the orchestrator's injected `GitHubClient` (via `context.github`)

**Rationale**:
- The `gh` CLI approach in `executeClarify()` runs inside a Claude subprocess where auth may not be available
- `context.github` uses authenticated Octokit, same as `StageCommentManager` and `LabelManager`
- Consistent with the orchestrator's existing patterns for GitHub interaction

**Alternatives considered**:
- `gh issue comment` via shell — fragile, auth-dependent, already proven unreliable (the bug)
- Direct Octokit import — unnecessary, client already injected

### 2. Markdown Parsing Approach

**Decision**: Simple regex/string parsing for `clarifications.md`

**Rationale**:
- The file format is well-defined (generated by speckit)
- Questions follow a predictable pattern: `### Q{n}: {topic}`, `**Answer**: *Pending*`
- No need for a full markdown AST parser

**Key patterns to match**:
- Question blocks: `### Q\d+: (.+)` to `---` delimiter
- Pending status: `**Answer**: \*Pending\*`
- Context/Question fields: `**Context**: (.+)`, `**Question**: (.+)`
- Options: `- [A-Z]\) .+`

### 3. Comment Format

**Decision**: Structured markdown matching the existing `clarifications.md` format

The comment should be immediately actionable — humans should be able to reply with answers directly in the issue thread.

## Implementation Patterns

### Dependency Injection Pattern

All worker classes follow the same DI pattern:
```typescript
class FooManager {
  constructor(
    private readonly github: GitHubClient,
    private readonly owner: string,
    private readonly repo: string,
    private readonly issueNumber: number,
    private readonly logger: Logger,
  ) {}
}
```

`ClarificationPoster` should follow this exact pattern.

### File Discovery Pattern

The spec directory convention is `specs/{issueNumber}-{slug}/`. Using glob `specs/{issueNumber}-*/clarifications.md` from `checkoutPath` reliably locates the file without needing to know the slug.
