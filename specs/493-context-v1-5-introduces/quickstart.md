# Quickstart: CLI Package Skeleton

**Feature**: #493 — CLI skeleton with placeholder subcommands and cluster registry

## Prerequisites

- Node.js >= 22
- pnpm (workspace package manager)

## Installation (development)

```bash
cd /workspaces/generacy
pnpm install
pnpm --filter @generacy-ai/generacy build
```

## Usage

### Version & Help

```bash
# Print version
npx generacy --version

# List all commands (existing + placeholders)
npx generacy --help
```

### Placeholder Commands

All v1.5 lifecycle commands are registered as placeholders:

```bash
npx generacy launch    # "Not yet implemented in this preview — landing in a future v1.5 phase 5 issue."
npx generacy up        # Same pattern
npx generacy stop
npx generacy down
npx generacy destroy
npx generacy status
npx generacy update    # phase 7
npx generacy open      # phase 6
npx generacy claude-login  # phase 6
npx generacy deploy    # phase 10
npx generacy rebuild   # phase 7
```

Each prints a friendly message with the v1.5 phase where the real implementation lands, then exits 0.

### Existing Commands (unchanged)

```bash
npx generacy run <workflow.yaml>    # Execute a workflow
npx generacy orchestrator           # Start orchestrator server
npx generacy validate               # Validate config
npx generacy doctor                 # Run diagnostics
npx generacy init                   # Initialize a project
```

### Logging

```bash
# Quiet mode (suppress info logs)
npx generacy --log-level silent status

# Debug mode
npx generacy --log-level debug status

# JSON output (no pretty printing)
npx generacy --no-pretty status
```

### Error Handling

```bash
# Normal mode: user-friendly error messages, no stack traces
npx generacy some-broken-command

# Debug mode: include stack traces
DEBUG=1 npx generacy some-broken-command
```

## Registry Helper (library API)

The cluster registry module is a library for use by future CLI commands:

```typescript
import {
  loadRegistry,
  saveRegistry,
  addCluster,
  removeCluster,
  findClusterByCwd,
} from '@generacy-ai/generacy/registry';

// Load the registry (creates empty if missing)
const registry = await loadRegistry();

// Add a cluster
await addCluster({
  id: 'clst_abc123',
  name: 'my-project',
  path: '/home/user/my-project',
  cloudUrl: 'https://api.generacy.ai',
  lastSeen: new Date().toISOString(),
});

// Find cluster for current directory
const cluster = await findClusterByCwd();
if (cluster) {
  console.log(`Found cluster: ${cluster.name}`);
}

// Remove a cluster
await removeCluster('clst_abc123');
```

## Troubleshooting

### "generacy requires Node.js 22 or later"

You're running an older Node version. Install Node 22+:
- https://nodejs.org/en/download
- Or via nvm: `nvm install 22 && nvm use 22`

### Command prints "not yet implemented"

This is expected. Placeholder commands will be replaced with real implementations in future v1.5 phase issues.

### Registry file is empty or corrupt

The registry at `~/.generacy/clusters.json` auto-recovers: if the file is missing or invalid JSON, `loadRegistry()` returns an empty registry. The next `saveRegistry()` call writes a valid file.
