## Implementation Approach

### Step 1: Extend the interface (`types.ts`)

Add `uid?: number` and `gid?: number` to the options parameter of `ProcessFactory.spawn`:

```typescript
export interface ProcessFactory {
  spawn(
    command: string,
    args: string[],
    options: { cwd: string; env: Record<string, string>; signal?: AbortSignal; uid?: number; gid?: number },
  ): ChildProcessHandle;
}
```

### Step 2: Update `defaultProcessFactory` (`claude-cli-worker.ts`)

Forward uid/gid to `child_process.spawn` options, using spread to only include them when defined:

```typescript
const child: ChildProcess = spawn(command, args, {
  cwd: options.cwd,
  env: { ...process.env, ...options.env },
  stdio: ['ignore', 'pipe', 'pipe'],
  ...(options.uid !== undefined && { uid: options.uid }),
  ...(options.gid !== undefined && { gid: options.gid }),
});
```

### Step 3: Update `conversationProcessFactory` (`process-factory.ts`)

Same pattern as Step 2 but with `['pipe', 'pipe', 'pipe']` stdio.

### Step 4: Add unit tests

- Test that when `uid`/`gid` are provided, they appear in `child_process.spawn` options
- Test that when `uid`/`gid` are omitted, they are not present in spawn options
- Cover both factory implementations
