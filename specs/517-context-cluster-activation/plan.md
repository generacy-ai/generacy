# Implementation Plan: Activation Poll Response cloud_url Fix

**Feature**: Fix activation client's poll response schema to accept `cloud_url` and persist it for boot-time re-use
**Branch**: `517-context-cluster-activation`
**Status**: Complete

## Summary

The activation client's `PollResponseSchema` rejects valid cloud responses because it's missing `cloud_url`. Even with that fixed, the orchestrator doesn't propagate the cloud-returned URL through `ActivationResult` or use it to configure the relay on boot. This fix touches three layers:

1. **Schema fix** тАФ Add `cloud_url` to the approved variant of `PollResponseSchema` in `activation-client`
2. **Result propagation** тАФ Add optional `cloudUrl` to `ActivationResult`, populate it from both the device-flow path and the existing-key path
3. **Boot-time override** тАФ `server.ts` reads `cloudUrl` from activation result and overrides both `config.activation.cloudUrl` (HTTPS) and `config.relay.cloudUrl` (derived WSS)

## Technical Context

**Language/Version**: TypeScript, ESM, Node >= 20

### Phase 2: Persistence + Return (orchestrator activation)

### Phase 3: Boot-Time Override (orchestrator server.ts)

**FR-004** тАФ After activation completes, use `cloudUrl` to override both config targets.

File: `packages/orchestrator/src/server.ts:307-315`

```typescript
const activationResult = await activate({ ... });
config.relay.apiKey = activationResult.apiKey;
config.relay.clusterApiKeyId = activationResult.clusterApiKeyId;

// Override cloud URLs if activation returned a custom URL
if (activationResult.cloudUrl) {
  config.activation.cloudUrl = activationResult.cloudUrl;
  // Derive WSS relay URL: https://X тЖТ wss://X/relay, http://X тЖТ ws://X/relay
  const relayUrl = activationResult.cloudUrl
    .replace(/^https:/, 'wss:')
    .replace(/^http:/, 'ws:')
    .replace(/\/$/, '') + '/relay';
  config.relay.cloudUrl = relayUrl;
}
```

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Existing `cluster.json` files missing `cloud_url` | High (all pre-fix clusters) | Low | `cloudUrl` is optional in `ActivationResult`; orchestrator only overrides config when present |
| Cloud API doesn't return `cloud_url` on older versions | Low | Medium | Schema requires it; if cloud doesn't send it, activation fails (fail-closed is correct тАФ the schema and cloud must agree) |
| WSS derivation incorrect for edge-case URLs | Low | Medium | Simple scheme swap + `/relay` suffix; matches existing pattern in config defaults |
