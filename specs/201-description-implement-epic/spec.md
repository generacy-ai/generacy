## Description

Implement epic processing support in the orchestrator, allowing issues with `type:epic` label to generate and track child issues.

generacy#195

## Behavior

1. Epic follows standard flow through specify/plan/tasks phases
2. After tasks phase, agent runs `/speckit:taskstoissues` to create child issues
3. Each child issue gets the appropriate `process:*` trigger label
4. Epic gets `waiting-for:children-complete` label
5. Monitor tracks child issue completion (via `completed:validate` labels)
6. When all children complete, epic is marked complete

## Tracking

- Parent-child relationship stored in Redis or via issue references
- Progress comment updated on epic as children complete
- Epic PR created as rollup from epic branch after all children merge
