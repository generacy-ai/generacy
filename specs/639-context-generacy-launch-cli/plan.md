# Implementation Plan: Scoped Private-Registry Credentials for `generacy launch`

**Feature**: Extend `pullImage` to authenticate against private registries using cloud-issued credentials
**Branch**: `639-context-generacy-launch-cli`
**Status**: Complete

## Summary

Add scoped, ephemeral Docker registry authentication to the `generacy launch` CLI's `pullImage` function. When `LaunchConfig` includes `registryCredentials`, write a temporary `DOCKER_CONFIG` directory with the auth entry, use it for the pull subprocess only, and clean up afterward. When no credentials are present, behavior is unchanged (ambient Docker auth).

## Technical Context

**Language/Version**: TypeScript (ESM), Node >= 22
**Primary Dependencies**: `commander`, `zod`, `@clack/prompts`, `node:child_process`, `node:fs`
**Storage**: Ephemeral filesystem (`<projectDir>/.docker/config.json`)
**Testing**: Vitest (unit tests with mocked `execSync`)
**Target Platform**: macOS, Linux, Windows (CLI)
**Project Type**: Monorepo package (`packages/generacy`)
**Constraints**: Must not touch `~/.docker/config.json`; cleanup in `finally` block

## Key Design Decisions

1. **Scoped config directory**: `<projectDir>/.docker/` — colocated with project, avoids tmp cleanup issues, unlikely to conflict with user files.
2. **File mode 0600**: Restricts read access to the credential file.
3. **`finally` block cleanup**: `rm -rf <projectDir>/.docker` ensures removal on both success and failure paths.
4. **`DOCKER_CONFIG` env override**: Only applied to the child `execSync` call, never set globally.
5. **No-creds path unchanged**: When `registryCredentials` is absent/undefined, the existing code path runs without modification.
6. **Error parsing**: Inspect stderr for `401`/`unauthorized` and `404`/`not found` patterns to produce actionable messages.
