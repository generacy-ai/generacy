# Research: `--cloud-url` flag for CLI commands

## Technology Decisions

### 1. Shared helper location: `cli/utils/cloud-url.ts`

**Decision**: Place `resolveCloudUrl()` in `packages/generacy/src/cli/utils/` rather than `commands/cluster/`.

**Rationale**: The helper is used by both `launch` (not a cluster lifecycle command) and `deploy`. The `cli/utils/` directory already contains cross-command utilities (`browser.ts`, `cluster-context.ts`, `exec.ts`). This matches spec Q2 recommendation.

**Alternatives considered**:
- `commands/cluster/` — too tightly scoped; `launch` is not a cluster lifecycle command
- Inline in each command — violates DRY, precedence logic would diverge

### 2. Validation strategy: eager Zod validation

**Decision**: Validate the resolved URL with `z.string().url()` inside `resolveCloudUrl()`.

**Rationale**: Fail-fast at CLI entry gives a clear error message ("Invalid --cloud-url: must be a valid URL with scheme") rather than a cryptic HTTPS error deep in the fetch call. This matches spec Q1 recommendation and the existing pattern in `LaunchConfigSchema`.

**Alternatives considered**:
- Defer to HTTP client — poor UX, error message doesn't point to the flag
- Custom regex — Zod `.url()` already covers scheme validation and is used elsewhere

### 3. Deploy env-var gap

**Discovery**: `deploy` already has `--cloud-url` as a Commander option but does NOT check `GENERACY_CLOUD_URL` env var. Using the shared helper automatically adds env-var support, making deploy consistent with launch.

**Risk**: An existing user who sets `GENERACY_CLOUD_URL` and uses `deploy` without `--cloud-url` currently gets the hardcoded default. After this change, they'd get the env var value. This is the *correct* behavior and matches user expectations — categorized as a bug fix, not a breaking change.

### 4. No short flag alias

**Decision**: No `-u` or similar short alias for `--cloud-url`.

**Rationale**: Primary consumer is machine-generated copy-paste commands from the cloud UI. Verbose form is more readable and self-documenting. Matches spec assumption.

```

```

### 3-tier resolution pattern

```typescript
import { z } from 'zod';

const DEFAULT_CLOUD_URL = 'https://api.generacy.ai';
const CloudUrlSchema = z.string().url();

export function resolveCloudUrl(flagValue?: string): string {
  const url = flagValue ?? process.env['GENERACY_CLOUD_URL'] ?? DEFAULT_CLOUD_URL;
  const result = CloudUrlSchema.safeParse(url);
  if (!result.success) {
    throw new Error(`Invalid cloud URL "${url}": must be a valid URL with scheme (e.g. https://api.generacy.ai)`);
  }
  return result.data;
}
```
