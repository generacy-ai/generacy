# Quickstart: `--cloud-url` flag

## Usage

### Launch with non-prod cloud

```bash
npx generacy launch --claim=abc123 --cloud-url=https://api-staging.generacy.ai
```

### Deploy to BYO VM against staging

```bash
generacy deploy ssh://user@host --cloud-url=https://api-staging.generacy.ai
```

### Precedence

1. `--cloud-url` flag (highest)
2. `GENERACY_CLOUD_URL` env var
3. `https://api.generacy.ai` (default)

```bash
# Flag wins over env var
GENERACY_CLOUD_URL=https://old.example.com generacy launch --claim=x --cloud-url=https://new.example.com
# → uses https://new.example.com

# Env var used when no flag
GENERACY_CLOUD_URL=https://staging.example.com generacy launch --claim=x
# → uses https://staging.example.com

# Default when neither is set
generacy launch --claim=x
# → uses https://api.generacy.ai
```
