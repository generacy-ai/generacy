## User Stories

### US1: Queue a whole phase in one command

**As a** cockpit operator driving an epic through phases,
**I want** to assign every open issue in a named phase to the cluster account and apply `process:speckit-feature` in a single confirm-gated command,
**So that** I can hand a phase off to the cluster pipeline without manually touching each issue.

**Acceptance Criteria**:
- [ ] `generacy cockpit queue <phase>` enumerates exactly the issues belonging to the named phase (no siblings from other phases).
- [ ] Before any mutation, the operator sees the resolved list of issues and confirms (skippable with `--yes`).
- [ ] On confirm, each issue is assigned to the cluster account and labelled `process:speckit-feature`.
- [ ] Already-assigned or already-labelled issues are idempotent (no-op, no error).
- [ ] No mutation occurs if confirmation is declined or `<phase>` resolves to zero issues.
