## Core Entities

### QueueItem

Represents an issue enqueued for workflow processing.

```typescript
interface QueueItem {
  /** Repository owner */
  owner: string;
  /** Repository name */
  repo: string;
  /** Issue number */
  issueNumber: number;
  /** Workflow name parsed from label (e.g., "speckit-feature") */
  workflowName: string;
  /** Command type: "process" for new, "continue" for resume */
  command: 'process' | 'continue';
  /** Priority score (timestamp for FIFO, lower = higher priority) */
```

```typescript
}
```

## Redis Key Patterns

### Phase Tracker Deduplication

```
Key:    phase-tracker:{owner}:{repo}:{issueNumber}:{phase}
Value:  "1" (presence-only)
TTL:    86400 (24 hours)
```

Example:
```
phase-tracker:generacy-ai:generacy:196:speckit-feature → "1" (TTL 24h)
```

```typescript
  pollIntervalMs: z.number().int().min(5000).default(30000),
  /** GitHub webhook secret for signature verification */
  webhookSecret: z.string().optional(),
  /** Maximum concurrent GitHub API calls during polling */
  maxConcurrentPolls: z.number().int().min(1).max(20).default(5),

  adaptivePolling: z.boolean().default(true),

```

## Validation Rules

- `workflowName` must be non-empty after parsing from label prefix
- `owner` and `repo` must match a configured watched repository
- `issueNumber` must be a positive integer
- Webhook signature must be valid HMAC-SHA256 when secret is configured
- Label names must match expected prefixes (`process:`, `waiting-for:`, `completed:`)
