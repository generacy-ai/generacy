# Implementation Plan: Template Library

**Feature**: [TG-006] [P] Template Library
**Branch**: `051-tg-006-p-template`
**Status**: In Progress

## Summary

Implement a template library system for the Generacy VS Code extension that provides:
1. File-based starter workflow templates
2. Template selection UI with preview capability
3. Schema-validated templates

## Technical Context

| Aspect | Choice | Rationale |
|--------|--------|-----------|
| Template Format | YAML files | Consistent with workflow format |
| Storage Location | resources/templates/ | Standard VS Code extension resource location |
| Loading | Bundled with extension | No network dependency, instant access |
| Validation | JSON Schema | Existing validator infrastructure |

## Implementation Approach

### Task 1: Create Starter Workflow Templates

Create three YAML template files that conform to the workflow schema:

1. **basic.yaml** - Single phase with one step for getting started
2. **multi-phase.yaml** - Setup/build/deploy pattern for CI/CD
3. **with-triggers.yaml** - Webhook and schedule triggers for automation

Templates will include:
- Schema-compliant structure (name, version, phases, etc.)
- Helpful comments explaining each section
- Placeholder values that are easy to customize

### Task 2: Implement Template Selection Quick Pick UI

The quick pick already exists in `workflow.ts`. Refactor to:
- Load templates from file system instead of inline strings
- Add template metadata (description, use case)
- Keep the existing UX flow

### Task 3: Add Template Preview

Enhance the quick pick dialog to:
- Show template content preview when an item is selected
- Use VS Code's preview capabilities
- Allow inspection before final selection

## Dependencies

- Existing workflow schema (`schemas/workflow.schema.json`)
- Existing workflow commands (`src/commands/workflow.ts`)
- VS Code QuickPick API with onDidChangeActive
