## Phase 2.2 — Cloud Platform Buildout

### Summary
Wire the cluster-relay client into the orchestrator's startup sequence so that a running cluster automatically connects to generacy-cloud.

### Context
The relay package (issue 2.1) provides the WebSocket client. This issue integrates it into the orchestrator so the connection is automatic and relay messages are routed to the correct handlers.

### Requirements

**Startup integration**:
- On orchestrator startup, if `GENERACY_API_KEY` is configured, instantiate and connect the relay
- Read cluster metadata from `.generacy/cluster.yaml` (issue 1.8) and environment
- Relay connection is optional — orchestrator works fully without it (local-only mode)
- Log relay connection status (connected, disconnected, reconnecting)

**API request routing**:
- Incoming `api_request` messages from the relay are forwarded to the local Fastify server
- Use Fastify's `inject()` method to route requests internally (no HTTP overhead)
- Return responses through the relay as `api_response` messages
- This means ALL existing orchestrator API endpoints are automatically available via relay — no duplication needed

**Event forwarding**:
- Existing SSE events (workflow lifecycle, phase progress, errors) are forwarded through the relay
- Subscribe to the orchestrator's event bus and forward events as `event` messages
- Cloud subscribers receive the same events they would get from direct SSE connection

**Metadata reporting**:
- On connect/reconnect, report cluster metadata:
  - Worker count from cluster.yaml
  - Active workflow count from workflow service
  - Channel from cluster.yaml
  - Orchestrator package version
  - Git remotes from workspace repos
  - Process uptime
- Periodically refresh metadata (e.g., every 60s) to keep cloud dashboard current

**Graceful shutdown**:
- On SIGTERM/SIGINT, disconnect relay before shutting down
- Cloud receives disconnect event and updates cluster status to offline

### Technical Notes
- Orchestrator entry point: `packages/orchestrator/src/index.ts`
- Fastify `inject()` docs: https://fastify.dev/docs/latest/Guides/Testing/#inject
- Existing SSE implementation in `packages/orchestrator/src/routes/events.ts`
- cluster.yaml reading from issue 1.8

### Dependencies
- Issue 2.1 (@generacy-ai/cluster-relay package)
- Issue 1.8 (cluster.yaml for metadata)
