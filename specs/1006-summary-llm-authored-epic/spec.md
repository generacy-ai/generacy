## Summary

## Summary

An LLM-authored epic body that uses **H4 (`####`) phase headers** silently resolves **every** child ref to `__adhoc__` instead of its phase. Because the epic then has no populated phases, S8 never emits `phase-complete`, so `/cockpit:auto`'s D.8 phase-queue gate never fires and the run idles indefinitely (heartbeat draining empty). The failure is **silent** — no warning, no error — and presents to the operator as "cockpit auto is stuck / the heartbeat isn't queuing the next phase."

## Root cause

`packages/cockpit/src/resolver/parse-epic-body.ts` enforces a strict heading grammar:

- `### X` (H3) **opens** a phase — `HEADING_L3_RE` at [parse-epic-body.ts:5](../blob/HEAD/packages/cockpit/src/resolver/parse-epic-body.ts#L5), handled at L91-99.
- `#### X` (H4+) **closes** the current phase — `HEADING_L4_PLUS_RE` at [parse-epic-body.ts:6](../blob/HEAD/packages/cockpit/src/resolver/parse-epic-body.ts#L6), handled at L74-78. It is a *terminator*, never an opener.
- A task ref with no open phase → `__adhoc__` — L127-138.

So an epic body shaped like:

```
### Delivery phases        ← H3: opens an empty phase (no task items directly under it)
#### P1 — Scaffold          ← H4: CLOSES the phase
- [ ] owner/repo#2          ← no open phase → __adhoc__
#### P2 — Foundation        ← H4: (already closed)
- [ ] owner/repo#3
...
```

routes all task refs to `__adhoc__`. The `phases[]` array contains only empty phases (`Scope`, `Tech stack`, `Delivery phases`, …), none carrying refs.

## Live evidence — `christrudelpw/snappoll#1`

Epic body (via `gh issue view 1`):

```
17:### Delivery phases
20:#### P1 — Scaffold
21:- [ ] christrudelpw/snappoll#2 — Scaffold Next.js + Tailwind app
23:#### P2 — Foundation
24:- [ ] christrudelpw/snappoll#3 — Prisma + data model + db client
25:- [ ] christrudelpw/snappoll#4 — Core utils: ID generator + poll validation
27:#### P3 — Core functionality
28-31:- [ ] #5 / #6 / #7 / #8
33:#### P4 — Polish & delivery
34-38:- [ ] #9 … #13
```

`generacy cockpit status christrudelpw/snappoll#1 --json` → **all 12 children report `"phase":"__adhoc__"`**. Issues #2/#3/#4 completed & merged; #5–#13 sit `pending` (`type:feature`, no `process:*` label) and never queue. No `phase-complete` was ever emitted; `/cockpit:auto` idled with the 300s heartbeat draining empty batches.

## Why intermittent

The epic body is generated by the epic-authoring action (`generacy-ai`). Across ~7 identical-prompt runs of this app, the previous 6 emitted phase headers at `###`; this run nested them one level deeper (`### Delivery phases` wrapping `#### P1…P4`). Same visual/rendered structure, valid Markdown — but H4 headers hit the terminator branch. Nondeterministic heading depth from the generator vs. a strict resolver grammar.

## Impact

- `/cockpit:auto` on an affected epic never advances past the initially-queued issues; remaining work sits `pending` forever.
- Presents as a cockpit-auto / heartbeat problem, costing significant debugging time before the epic-body cause is found.

## Recommended fix (two-sided)

1. **Authoring (primary):** pin epic phase headers to `###` in the epic-authoring template/prompt. Do not emit `####` for phases (reserve H4+ for the intended terminator role).
2. **Resolver (defense-in-depth):** when the parse yields a non-empty `adhocRefs[]` **and** the body contains `####`-level headers whose text is phase-shaped (e.g. `#### P<n>`/`Phase`), push a `warnings[]` entry — e.g. `"N task refs fell to ad-hoc; phase headers must be '###', found '####'"`. The warnings channel already exists ([parse-epic-body.ts:15-33](../blob/HEAD/packages/cockpit/src/resolver/parse-epic-body.ts#L15-L33)); surfacing it turns a silent multi-hour stall into an immediate, actionable signal. Optionally, `/cockpit:auto`'s startup sweep should print a loud line when an epic resolves to **all-adhoc, zero populated phases** rather than idling quietly.

## Repro

Author an epic body with `### <section>` followed by `#### P1 …` phase headers and task-list refs under each; run `generacy cockpit status <epic> --json`; observe all refs report `phase: __adhoc__` and no `phase-complete` is emitted.

## Decisions from clarifications (Batch 1)

Locked-in choices from `clarifications.md` Batch 1 that constrain plan/implement:

- **FR-002 — phase-shaped `####` detector (Q1=C, refined)**: A `####` heading is "phase-shaped" iff its trimmed text matches `/^\s*P\d+\b/i` **OR** contains a word-boundaried case-insensitive `phase` (`/\bphase\b/i`). Word-boundary is load-bearing on the `phase` arm so `#### Rephrase …` does not match. Excludes `#### Notes` / `#### Follow-ups` (SC-002).
- **FR-003/FR-004 — stable warning marker (Q2=B)**: Tests assert via `toContain("phase headers must be '###'")`. Full sentence stays free to evolve (count, `found '####'` phrasing, punctuation). This substring must not appear in any other resolver rejection marker (grep audit per SC-006).
- **FR-009 surfacing scope (Q3=A, with Q4/Q5 caveat)**: This PR ships the fix **end-to-end**: resolver-side warning + surfacing on both the `cockpit status --json` envelope *and* the `cockpit_status` MCP tool return shape. No companion PR gap. **Do not** add an `allAdhocZeroPopulatedPhases: boolean` field — the richer degradation return-type is excluded by Out-of-Scope §5.
- **FR-009 surfacing channel (Q4=D)**: stderr line for human operators **plus** `warnings[]` in the `--json` envelope for `/cockpit:auto`'s sweep. No `degradation.kind`; the auto skill infers degradation from `warnings[].some(w => w.includes("phase headers must be '###'"))`.
- **FR-012 envelope shape (Q5=A)**: Add `warnings: string[]` to the `cockpit status --json` envelope AND the `cockpit_status` MCP tool return, in this PR. Sourced directly from `parsed.warnings`; empty array when clean; additive / non-breaking.

## Correctness constraint — FR-009 part (b) gating

FR-009 part (b) (the "loud signal for all-adhoc + zero-populated-phases") must be gated on **the Q1 detector**, not on a bare `phases.every(p => p.refs.length === 0) && adhocRefs.length > 0` predicate.

`[].every(...)` is vacuously `true`, so the bare predicate fires on legitimate flat-list bodies where `phases.length === 0` and every ref is adhoc — a mode `packages/cockpit/src/resolver/resolve.ts:57-63` documents as valid and supported. Gating on the bare predicate would fail SC-002 (false positive on legitimate flat-list epics).

Concrete gating rule: fire the loud signal iff **(a)** `phases.length > 0` **AND** **(b)** `phases.every(p => p.refs.length === 0)` **AND** **(c)** `adhocRefs.length > 0` **AND** **(d)** the body contains at least one `####` heading whose text matches the Q1=C detector.
