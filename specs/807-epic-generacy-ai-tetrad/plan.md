# Implementation Plan: Collapse cockpit CLI surface to rev 3 catalog

**Feature**: Epic: generacy-ai/tetrad-development#85 | Phase: S3 | Tier: v1-simplification | Issue: G-S3

**Status**: Complete
**Spec**: [spec.md](./spec.md) | **Date**: 2026-07-06

## Summary

Merge three cockpit verbs (`state`, `clarify-context`, `review-context`) into a single `generacy cockpit context <issue>` verb that classifies the issue's current `waiting-for:*` gate and emits the exact bundle each gate needs:

| Gate label | Bundle emitted |
|---|---|
| `waiting-for:clarification` | `{issue, gate, clarificationComment, spec, plan, codeReferences}` (matches today's `clarify-context`) |
| `waiting-for:implementation-review` | `{issue, gate, pr, diff, diffTruncated, checks}` (matches today's `review-context`) |
| `waiting-for:spec-review` \| `waiting-for:plan-review` \| `waiting-for:tasks-review` | `{issue, gate, artifacts: {spec, plan, tasks}}` where each is `{path, body} \| null` (new shape defined in this spec) |
| any other, or missing | exit 3 (gate refusal); if PR-scoped gate has no resolvable PR, also exit 3 |

Additionally: fold the CLI-local `gh-ext.ts` (`CockpitGh`, ~306 LOC) into the shared engine `GhWrapper` at `packages/cockpit/src/gh/wrapper.ts`, and collapse the ref/scope resolver modules (`issue-ref.ts` + `shared/resolve-context.ts`; `shared/scoping.ts` referenced by the spec does not currently exist as a source file) into one module.

`advance` and `merge` command behavior are unchanged and their existing tests must pass unmodified.

## Implementation Phases

### Phase 0 ‚Äî GhWrapper fold-in (unblocks everything)

1. Extend `GhWrapper` interface in `packages/cockpit/src/gh/wrapper.ts` with the eight methods currently only on `CockpitGh` that have no `GhWrapper` equivalent:
   - `fetchIssueLabels(repo, n)` ‚Äî thin variant of `getIssue` returning only labels
   - `fetchIssueState(repo, n)` ‚Äî labels + state + closedAt + assignees + title
   - `postIssueComment(repo, n, body)` ‚Äî returns `{ url }`
   - `addAssignees(repo, n, logins)` ‚Äî iterates `--add-assignee`
   - `fetchIssueTimeline(repo, n)` ‚Äî paginated `gh api`
   - `fetchIssueComments(repo, n)`
   - `getCurrentUser()`
   - `findOpenPrForBranch(repo, branch)` ‚Äî replaces `resolveIssueToPRRef` for branch-based lookups
   - `prDiffNames(repo, prNumber)` ‚Äî `--name-only`
   - `prDiffPatch(repo, prNumber)` ‚Äî `--patch` (superseded by `getPullRequestDetail`'s cap logic; kept for `code-references.ts`)
2. Overlapping methods reuse existing implementations:
   - Single-label `addLabel`/`removeLabel` become one-liners over existing plural `addLabels`/`removeLabels`.
   - `fetchIssueLabels` / `fetchIssueState` are new JSON projections of `getIssue`.
3. Delete `gh-ext.ts`. Migrate every `CockpitGh` call-site:
   - `state.ts`, `clarify-context.ts`, `advance.ts`, `merge.ts`, `code-references.ts`, `clarification-comment-finder.ts`, `watch.ts` ‚Äî currently type `CockpitGh` gets replaced with `GhWrapper` from `@generacy-ai/cockpit`.
4. Update tests to stub `GhWrapper` shape (add fields to helpers/fixtures).

### Phase 1 ‚Äî Resolver collapse

1. Create `packages/generacy/src/cli/commands/cockpit/resolver.ts` exporting:
   - `IssueRef` (owner/repo/number/nwo) ‚Äî verbatim from today.
   - `parseIssueRef(input, cwd?)` ‚Äî accepts `owner/repo#N`, full URL, or bare number (only when cwd inference succeeds ‚Äî see FR-005 clarification).
   - `resolveIssueContext({ issue, cwd? })` ‚Äî returns `{ ref, repo, gh: GhWrapper }`. Internally allows an override `repo?: string` for programmatic callers (spec Q5 explicitly permits an internal-only override).
2. Delete `issue-ref.ts` and `shared/resolve-context.ts`. Migrate imports:
   - `parseIssueRef` importers: `state.ts` (‚Üí deleted), `clarify-context.ts` (‚Üí deleted), `advance.ts`, `merge.ts` ‚Äî swap to `resolver.ts`.
   - `resolveContext` importers: `review-context.ts` (‚Üí deleted). No other callers.
3. Update `resolver.test.ts` (merge of `issue-ref.test.ts` + inference-from-origin scenarios).

### Phase 2 ‚Äî Unified `context` verb

1. Create `context.ts` with the three bundle branches. Pseudocode outline:

    ```ts
    async function runContext(issueArg, deps) {
      const ref = parseIssueRef(issueArg);
      const gh = deps.gh ?? new GhCliWrapper(runner);
      const labels = await gh.fetchIssueLabels(ref.nwo, ref.number);

      const gate = firstGate(labels);           // per WAITING_PIPELINE_ORDER
      if (gate == null) throw new CockpitExit(3, `no waiting-for:* label on ${ref.nwo}#${ref.number} (labels: ${labels.join(',')})`);
      if (gate === 'completed:validate') throw new CockpitExit(3, '... use `cockpit merge`');

      switch (gate) {
        case 'waiting-for:clarification':          return emitClarificationBundle(ref, gh, ...);
        case 'waiting-for:implementation-review':  return emitImplementationReviewBundle(ref, gh, ...);
        case 'waiting-for:spec-review':
        case 'waiting-for:plan-review':
        case 'waiting-for:tasks-review':           return emitArtifactPathsBundle(ref, gate, ...);
        default:                                    throw new CockpitExit(3, `unsupported gate: ${gate}`);
      }
    }
    ```

2. Reuse `buildReviewContextPayload` from `shared/review-context-json.ts` for the implementation-review bundle (avoid re-implementing).
3. Reuse `findClarificationComment`, `gatherCodeReferences`, and the spec/plan-reader logic from today's `clarify-context.ts` for the clarification bundle.
4. Implement `readArtifactBundle(cwd, branch, issueNumber)` for the artifact-paths bundle ‚Äî always emits `spec`, `plan`, `tasks` (each `{path, body} | null`).
5. Wire `contextCommand()` into `index.ts`; delete the three replaced `addCommand` calls.
6. Preserve `CockpitExit` failure surface exactly ‚Äî no diagnostic wording change beyond gate-refusal messages.
