# Quickstart: Core Credential Type Plugins (#463)

## Prerequisites

- Node.js 18+
- pnpm
- `@generacy-ai/credhelper` and `@generacy-ai/credhelper-daemon` packages built

## Installation

```bash
pnpm install
pnpm -F @generacy-ai/credhelper build
pnpm -F @generacy-ai/credhelper-daemon build
```

## Plugin Usage

### Registering Core Plugins

Core plugins are statically registered — no discovery pipeline needed:

```typescript
import { CORE_PLUGINS } from './plugins/core';

// Register all core plugins
const registry = new Map<string, CredentialTypePlugin>();
for (const plugin of CORE_PLUGINS) {
  registry.set(plugin.type, plugin);
}
```

## Troubleshooting

### Plugin not found
Ensure the plugin is listed in `CORE_PLUGINS` array in `packages/credhelper-daemon/src/plugins/core/index.ts`.

### Schema validation errors
Check that your `credentials.yaml` matches the plugin's `credentialSchema`. Run validation:
```typescript
import { githubAppPlugin } from './plugins/core/github-app';
githubAppPlugin.credentialSchema.parse(yourConfig); // throws ZodError if invalid
```

### Mint failures
- **github-app**: Verify `appId`, `installationId`, and that the private key in the backend is a valid PEM
- **gcp-service-account**: Verify `serviceAccountEmail` exists and the base credentials have `iam.serviceAccounts.getAccessToken` permission
- **aws-sts**: Verify `roleArn` exists, trust policy allows assumption, and `externalId` matches if required

### env-passthrough returns empty
The env var specified by `backendKey` must be set in the daemon's environment. Check with `echo $GITHUB_TOKEN` (or whatever the backendKey is).
