# Implementation Plan: Fix vscode-tunnel-manager CONNECTED_PATTERN

**Feature**: Update `CONNECTED_PATTERN` regex to match `code` CLI 1.95.3 output and emit `error` event on unexpected process exit
**Branch**: `606-symptoms-user-clicks-vs`
**Status**: Complete

## Summary

Two fixes in `packages/control-plane/src/services/vscode-tunnel-manager.ts`:

1. **CONNECTED_PATTERN regex** (FR-001): The current pattern `/is connected|tunnel is ready/i` doesn't match `code` CLI 1.95.3's actual output (`"Open this link in your browser https://vscode.dev/tunnel/<name>/..."`). Add `https://vscode.dev/tunnel/[\w-]+` as the primary match, keeping old alternatives as fallback.

2. **Exit handler error emission** (FR-002): When `code tunnel` exits during `starting` or `authorization_pending` states, no event is emitted. Add a branch to emit an `error` event with exit code and last 20 stdout lines.

3. **Tunnel URL extraction** (FR-003, P2): Extract the `vscode.dev` tunnel URL from the connected stdout line and include it in the `connected` event payload.

## Implementation Details

### Change 1: Update CONNECTED_PATTERN (FR-001)

**File**: `packages/control-plane/src/services/vscode-tunnel-manager.ts:40`

```diff
- const CONNECTED_PATTERN = /is connected|tunnel is ready/i;
+ const CONNECTED_PATTERN = /https:\/\/vscode\.dev\/tunnel\/[\w-]+|is connected|tunnel is ready/i;
```

The new primary match targets the `https://vscode.dev/tunnel/<name>/` URL that `code` CLI 1.95.3 emits when the tunnel is ready. Old alternatives remain as fallback for future CLI version changes.

### Change 2: Emit error on unexpected exit (FR-002)

**File**: `packages/control-plane/src/services/vscode-tunnel-manager.ts:102-119`

Replace the `else { this.status = 'stopped'; }` branch with a check for `authorization_pending` and `starting` states. Emit an `error` event with exit code and last 20 stdout lines.

### Change 3: Extract tunnel URL (FR-003)

**File**: `packages/control-plane/src/services/vscode-tunnel-manager.ts:219-225`

Add a regex capture for the full `https://vscode.dev/tunnel/<name>/...` URL from the connected stdout line. Include it as `tunnelUrl` in the `connected` event payload. Requires adding `tunnelUrl?: string` to the `VsCodeTunnelEvent` interface.
