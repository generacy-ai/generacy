## Summary

The `@generacy-ai/templates` package (`packages/templates/`) contains Handlebars-based devcontainer templates (Dockerfile.hbs, docker-compose.yml.hbs, etc.) that have been **fully superseded** by the external [`cluster-templates`](https://github.com/generacy-ai/cluster-templates) repository.

The onboarding worker in `generacy-cloud` now exclusively fetches devcontainer files from `cluster-templates` via GitHub API — the Handlebars templates are no longer in the active code path.

## What to clean up

### 1. Remove the templates package
- Delete `packages/templates/` entirely
- Remove from `pnpm-workspace.yaml` if listed
- Remove any workspace dependency references

### 2. Migrate the `ClusterVariant` type
The only remaining import from this package is the `ClusterVariant` type, used in:
- `packages/generacy/src/cli/commands/init/types.ts`
- `packages/generacy/src/cli/commands/init/summary.ts`

Move this type to the CLI package or a shared types location.

## Context

The transition happened when the `cluster-templates` repo was set up to provide a Docker Compose-based development cluster (orchestrator + workers) as the standard onboarding path. The worker service in `generacy-cloud` (`services/worker/src/lib/cluster-templates.ts`) now fetches templates directly from that repo at a configurable ref.
