# Implementation Plan: CLI Cluster Lifecycle Commands

**Feature**: Implement cluster-lifecycle subcommands (`up`, `stop`, `down`, `destroy`, `status`, `update`) for the `generacy` CLI
**Branch**: `494-context-implement-cluster`
**Status**: Complete

## Summary

Add six cluster lifecycle commands to the existing `generacy` CLI at `packages/generacy/src/cli/commands/`. Each command wraps `docker compose` operations, using a shared `ClusterContext` resolved from `.generacy/cluster.yaml`, `.generacy/cluster.json`, and `.generacy/docker-compose.yml`. A host-level cluster registry at `~/.generacy/clusters.json` tracks all known clusters for the `status` command.

The implementation follows the existing command patterns (Commander.js v12, Pino logger, `@clack/prompts` for interactive prompts, `vitest` for testing) and reuses the `exec`/`execSafe` utilities for Docker Compose invocations.

## Key Design Decisions

### 1. Shared helpers in `commands/cluster/`

All six commands share context resolution, compose argument building, registry management, and Docker checks. These live in `commands/cluster/` rather than `utils/` because they are specific to cluster lifecycle, not general CLI utilities.

### 2. `execSafe` for Docker Compose

All Docker Compose invocations use `execSafe()` to capture stdout/stderr without throwing, allowing user-friendly error handling (e.g., "Docker is not running" instead of a stack trace).

### 3. Registry as JSON array

`~/.generacy/clusters.json` is a JSON array of `RegistryEntry` objects. Atomic writes via temp + rename pattern (consistent with orchestrator activation persistence). No locking needed since only one CLI process writes at a time.

### 4. Fallback cluster ID

If `.generacy/cluster.json` doesn't exist (pre-activation), use `path.basename(projectRoot)` as the Compose project name and emit a warning. This allows `up` to work before activation completes.

### 5. `status` uses `docker compose ps --format json`

Docker Compose v2 supports `--format json` output, which we parse to determine container states. This avoids fragile text parsing.

### 6. `destroy` confirmation via `@clack/prompts`

Uses `p.confirm()` from `@clack/prompts` consistent with the `init` command's conflict resolution pattern. `--yes` flag bypasses the prompt.
