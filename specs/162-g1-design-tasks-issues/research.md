# Research: Tasks-to-Issues Workflow

## Technology Decisions

### Workflow Engine Format

**Decision**: Use Generacy's native YAML workflow format

**Rationale**:
- Consistent with other Generacy workflows
- Supports step-based execution with conditions
- Integrates with Humancy for human-in-the-loop patterns
- Expression syntax (`${{ }}`) for variable interpolation

### Human-in-the-Loop Pattern

**Decision**: Use `humancy.request_review` action with no timeout

**Rationale**:
- Human oversight is critical for issue creation (can't be undone easily)
- Indefinite wait ensures no accidental approvals
- Consistent with fail-safe design principle

**Alternatives Considered**:
- Auto-approve after timeout: Rejected due to risk of unreviewed issues
- Escalation: Adds complexity, not needed for initial implementation

### Provider Detection

**Decision**: Auto-detect from repository configuration

**Implementation**:
```yaml
# Detection logic (pseudo-code):
# 1. Check for .github/ directory → GitHub
# 2. Check for .jira/ config → Jira
# 3. Check for shortcut.json → Shortcut
# 4. Default to 'local' (file-based)
```

**Rationale**:
- Most repos already have provider hints in their structure
- Reduces required configuration
- Can always be overridden explicitly

### Error Handling

**Decision**: Fail-fast approach

**Rationale**:
- Issue creation is not idempotent (duplicates are problematic)
- Partial completion leaves inconsistent state
- Clear failure is better than silent partial success

## Implementation Patterns

### Step Conditions

```yaml
- id: create
  if: ${{ steps.review.approved && !inputs.dry_run }}
```

The `if` condition ensures:
1. Human review was approved
2. Dry-run mode is not enabled

### Output Chaining

Steps can reference outputs from previous steps:
```yaml
action: agent.invoke
with:
  prompt: |
    Read tasks at ${{ steps.parse_tasks.outputs.tasksFile }}
```

### Humancy Integration

```yaml
action: humancy.request_review
with:
  artifact: |
    ## Issues to Create
    ${{ steps.preview.outputs.summary }}
  context: Review the planned issues before creation
```

The `artifact` field contains the content to review, and `context` provides reviewer guidance.
