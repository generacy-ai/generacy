# Data Model: Tasks-to-Issues Workflow

## Core Entities

### WorkflowInput

Input parameters for the workflow.

```typescript
interface WorkflowInput {
  feature_dir: string;           // Required: path to feature directory
  grouping?: GroupingStrategy;   // Optional: how to group tasks into issues
  provider?: ProviderType;       // Optional: backlog provider override
  dry_run?: boolean;             // Optional: preview only mode (default: false)
}

type GroupingStrategy = 'per-task' | 'per-story' | 'per-phase';
type ProviderType = 'github' | 'jira' | 'shortcut' | 'local';
```

### IssueOutput

Standard output format for created issues.

```typescript
interface IssueOutput {
  id: string;          // Issue identifier (number for GitHub, key for Jira)
  url: string;         // Full URL to the issue
  title: string;       // Issue title
  provider: ProviderType;  // Provider used to create the issue
}

interface WorkflowOutput {
  issues: IssueOutput[];  // Array of created issues
}
```
