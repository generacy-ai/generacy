# Quickstart: Tasks-to-Issues Workflow

## Overview

Convert tasks from `tasks.md` into backlog issues with human review.

## Usage

### Basic Usage

```bash
# Invoke via Generacy CLI or UI
generacy workflow run tasks-to-issues \
  --input feature_dir=specs/my-feature
```

### With Options

```bash
# Group tasks by story, target Jira
generacy workflow run tasks-to-issues \
  --input feature_dir=specs/my-feature \
  --input grouping=per-story \
  --input provider=jira
```

### Dry Run (Preview)

```bash
# Preview issues without creating them
generacy workflow run tasks-to-issues \
  --input feature_dir=specs/my-feature \
  --input dry_run=true
```

## Inputs

| Input | Required | Default | Description |
|-------|----------|---------|-------------|
| `feature_dir` | Yes | - | Path to feature directory with tasks.md |
| `grouping` | No | `per-task` | `per-task`, `per-story`, or `per-phase` |
| `provider` | No | auto-detect | `github`, `jira`, `shortcut`, or `local` |
| `dry_run` | No | `false` | Preview only, don't create issues |

## Workflow Steps

1. **Parse tasks.md** - Resolves file paths
2. **Validate dependencies** - Checks for circular references
3. **Generate preview** - Creates issue preview
4. **Human review** - Waits for approval via Humancy
5. **Create issues** - Creates issues in backlog system

## Triggers

- **Manual**: Invoke from Generacy UI or CLI
- **Workflow step**: Use as a step in larger workflows
- **Event**: Trigger on tasks.md file commit

## Troubleshooting

### "tasks.md not found"
Ensure the `feature_dir` path is correct and contains a `tasks.md` file.

### "Circular dependency detected"
Review your tasks.md for dependency cycles. Each task should only depend on tasks defined earlier.

### Review not appearing
The workflow waits indefinitely for human approval. Check your Humancy notification settings.

### Provider detection failed
Explicitly set the `provider` input if auto-detection doesn't work for your setup.
