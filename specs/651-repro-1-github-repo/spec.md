# Bug Fix: CLI scaffolder hardcodes REPO_BRANCH=main

**Branch**: `651-repro-1-github-repo` | **Date**: 2026-05-19 | **Status**: Draft | **Issue**: [#651](https://github.com/generacy-ai/generacy/issues/651)

## Summary

The CLI scaffolder in `packages/generacy/src/cli/commands/cluster/scaffolder.ts` hardcodes `REPO_BRANCH=main` in the generated `.env` file when no branch is specified. This causes post-activation clone failures for any GitHub repo whose default branch is not `main` (e.g., `develop`, `master`, `trunk`). The fix removes the `'main'` fallback so that when no branch is specified, `REPO_BRANCH` is omitted from the env file, allowing `git clone` to use the repo's HEAD (its actual default branch).

## Root Cause

`scaffolder.ts` line ~279 defaults `repoBranch` to `'main'`:
```typescript
const repoBranch = input.repoBranch ?? 'main';
```
The cloud's `LaunchConfig` doesn't include a branch field, and the CLI launch flow doesn't pass `repoBranch` to the scaffolder — so the `?? 'main'` fallback always wins.
