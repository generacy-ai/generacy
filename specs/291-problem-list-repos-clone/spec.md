# Feature Specification: ## Problem

The list of repos to clone and monitor is defined in **5 separate locations** across 2 repos, using 2 different formats

**Branch**: `291-problem-list-repos-clone` | **Date**: 2026-03-04 | **Status**: Draft

## Summary

## Problem

The list of repos to clone and monitor is defined in **5 separate locations** across 2 repos, using 2 different formats. Adding or removing a repo requires coordinated edits that easily drift out of sync (e.g., `contracts` was in one list but not another, `cluster-templates` was missing from `DEFAULT_REPOS`, and `agent.env.template` had a typo turning a comma into a slash).

### Current locations

| Location | Format | Purpose |
|----------|--------|---------|
| `packages/generacy/src/cli/commands/setup/workspace.ts:27` | Bare names (`"generacy"`) | `DEFAULT_REPOS` for `generacy setup workspace` |
| `packages/generacy/src/cli/commands/orchestrator.ts:206` | `owner/repo` from env | Parses `MONITORED_REPOS` for label monitor init |
| `packages/orchestrator/src/config/loader.ts:108` | `owner/repo` from env | Parses `MONITORED_REPOS` / `ORCHESTRATOR_REPOSITORIES` |
| `packages/generacy/src/orchestrator/job-handler.ts:579` | `owner/repo` from env | Maps GitHub repos → local `/workspaces/` paths |
| `tetrad-development/.devcontainer/agent.env.template:14` | `owner/repo` csv | Template for `MONITORED_REPOS` env var |

### Format inconsistency
- Workspace setup uses **bare repo names**: `"generacy"`
- Everything else uses **owner/repo**: `"generacy-ai/generacy"`

## Proposed Solution

Create a `.generacy/config.yaml` in the project's primary repo (tetrad-development) as the single source of truth. All consumers read from this file, with env vars and CLI flags as overrides.

### Config file location
`.generacy/config.yaml` (alongside the existing `.generacy/speckit-*.yaml` workflow files)

### Example structure
```yaml
workspace:
  org: generacy-ai
  branch: develop
  repos:
    - name: tetrad-development
      monitor: true
    - name: cluster-templates
      monitor: true
    - name: latency
      monitor: true
    - name: agency
      monitor: true
    - name: generacy
      monitor: true
    - name: humancy
      monitor: true
    - name: generacy-cloud
      monitor: true
    - name: humancy-cloud
      monitor: true
```

### Changes needed

1. **Create config loader** — New utility in generacy that reads `.generacy/config.yaml` from the workspace root
2. **Update `generacy setup workspace`** — Read repos from config file instead of `DEFAULT_REPOS` constant; keep `--repos` CLI and `REPOS` env as overrides
3. **Update orchestrator startup** (`orchestrator.ts`) — Read monitored repos from config file; keep `--monitored-repos` CLI and `MONITORED_REPOS` env as overrides
4. **Update `loader.ts`** — Support reading `repositories` from the centralized config
5. **Update `job-handler.ts`** — Use the same config source instead of re-parsing `MONITORED_REPOS` independently
6. **Update `agent.env.template`** — Document that `MONITORED_REPOS` is an override, not the source of truth
7. **Remove `DEFAULT_REPOS` constant** from `workspace.ts`

### Override priority (preserved)
1. CLI flags (`--repos`, `--monitored-repos`)
2. Environment variables (`REPOS`, `MONITORED_REPOS`)
3. `.generacy/config.yaml` file
4. Built-in defaults (empty — config file is required)

### Notes
- The `.generacy/` directory already exists in tetrad-development with workflow files (`speckit-feature.yaml`, `speckit-bugfix.yaml`)
- The orchestrator config loader already supports YAML files (`orchestrator.yaml`) but none exists — this could be unified
- Companion issue in tetrad-development tracks creating the actual config file
