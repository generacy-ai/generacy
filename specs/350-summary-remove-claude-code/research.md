# Research: Remove Claude Marketplace Plugin Install

**Feature**: 350-summary-remove-claude-code | **Date**: 2026-03-07

## Technology Decisions

### Decision 1: Package Resolution Strategy

**Chosen**: Path probing with `existsSync` over `require.resolve`

**Rationale**: The current codebase already uses `existsSync` + `join` for path resolution (see `resolveNpmGlobalRoot()` and agency CLI resolution). Using `require.resolve` would require a try/catch wrapper and may behave unexpectedly with ESM packages. Path probing is simpler, debuggable, and consistent with existing patterns.

**Alternatives Considered**:
- `require.resolve('@generacy-ai/agency-plugin-spec-kit/commands')` — cleaner but unreliable with ESM, may resolve to package entry point instead of directory
- `import.meta.resolve` — ESM-only, not available in all Node versions used
- `execSafe('npm ls -g --parseable @generacy-ai/agency-plugin-spec-kit')` — too slow for a setup command

### Decision 2: Package Name Change

**Chosen**: Use `@generacy-ai/agency-plugin-spec-kit` (npm package name)

**Rationale**: The spec explicitly requires sourcing commands from this package. The current fallback uses `@generacy-ai/agency/commands/` which is the monorepo root package, not the dedicated plugin package. The dedicated package will bundle commands as part of the dependency (generacy-ai/agency repo work).

### Decision 3: Remove vs Disable Marketplace Code

**Chosen**: Remove entirely

**Rationale**: The spec says "Remove" (FR-005), not "disable" or "feature flag". Dead code adds maintenance burden. The marketplace approach is fundamentally unreliable (doesn't materialize `.md` files consistently), so there's no reason to keep it behind a flag.

### Decision 4: Retain Source Directory Fallback?

**Chosen**: Remove the source directory fallback (`config.agencyDir` path)

**Rationale**: The source directory fallback (`{agencyDir}/packages/claude-plugin-agency-spec-kit/commands/`) is only useful during development when the agency repo is cloned. Once the npm package bundles command files, local workspace `node_modules` will contain the same files via the workspace link. The two-tier resolution (local node_modules → npm global) covers both development and production scenarios.

## Implementation Patterns

### Pattern: Two-Tier Resolution

```typescript
function resolveSpeckitCommandsDir(config: BuildConfig): string | null {
  // Tier 1: Local workspace node_modules
  const localPaths = [
    join(config.workspaceRoot, 'node_modules', '@generacy-ai', 'agency-plugin-spec-kit', 'commands'),
    join(config.agencyDir, 'node_modules', '@generacy-ai', 'agency-plugin-spec-kit', 'commands'),
  ];
  for (const p of localPaths) {
    if (existsSync(p)) return p;
  }

  // Tier 2: npm global
  const globalRoot = resolveNpmGlobalRoot();
  if (globalRoot) {
    const globalPath = join(globalRoot, '@generacy-ai', 'agency-plugin-spec-kit', 'commands');
    if (existsSync(globalPath)) return globalPath;
  }

  return null;
}
```
