# Implementation Plan: Boot-render uploaded file blobs on container recreate

**Feature**: Uploaded app-config files lost on `docker compose down && up`
**Branch**: `637-summary-when-user-uploads`
**Status**: Complete

## Summary

`AppConfigSecretEnvStore` already re-renders encrypted env vars at boot (#632). The parallel for files was never implemented: `AppConfigFileStore` stores encrypted blobs persistently in `ClusterLocalBackend` (survives restart) but the decoded file at `mountPath` lands on the container's writable overlay layer and is wiped on recreate.

**Fix**: Add `renderAll(readManifest)` to `AppConfigFileStore` that walks `values.yaml.files` metadata, fetches each encrypted blob from the backend, resolves the current `mountPath` from the manifest, validates against the denylist, and atomically writes to disk. Called from `bin/control-plane.ts` immediately after the existing secret env render.

## Design Decisions

### D1: renderAll() lives on AppConfigFileStore (not a new class)

The file store already owns the backend reference, metadata access, and atomic write logic. Adding `renderAll()` here follows the same pattern as `AppConfigSecretEnvStore.renderAll()`.

### D2: Manifest passed as a callback, not inline

`readManifest()` is currently private to `app-config.ts`. Rather than importing it (and its dependency chain: `resolveGeneracyDir`, yaml parsing), `renderAll()` accepts a callback `() => Promise<AppConfig | null>`. This keeps the store testable and avoids coupling.

### D3: isPathDenied() extracted as shared helper

Currently inline in `app-config.ts`. Both the route handler (`handlePostFile`) and the new `renderAll()` need it. Export from `app-config.ts` directly (no new file — minimal change).

### D4: Atomic write helper extracted from setFile()

`setFile()` already has the temp+datasync+rename+mkdir pattern (lines 88-99). Extract into a private `atomicWriteFile(absPath, data)` method shared by both `setFile()` and `renderAll()`. No new file needed.

### D5: RenderResult type reused from AppConfigSecretEnvStore

Import and reuse `RenderResult { rendered: string[]; failed: string[] }` — identical semantics. Or define locally in file-store if import creates circular dependency.

### D6: Boot-render surfaces in initResult (same pattern as #624)

File render failures added to `initResult.warnings[]` array, matching the existing secret env render pattern at `bin/control-plane.ts:141-152`.

### Step 2: Add `renderAll()` and `atomicWriteFile()` to AppConfigFileStore

```typescript
// New method on AppConfigFileStore
async renderAll(
  readManifest: () => Promise<AppConfig | null>,
): Promise<{ rendered: string[]; failed: string[] }> {
  if (this.status === 'disabled') {
    return { rendered: [], failed: [] };
  }

  const meta = await this.readMetadata();
  const manifest = await readManifest();
  const rendered: string[] = [];
  const failed: string[] = [];

  const manifestFiles = new Map(
    (manifest?.files ?? []).map(f => [f.id, f]),
  );

  for (const id of Object.keys(meta.files)) {
    // Skip if file id not in current manifest (orphaned blob)
    const fileEntry = manifestFiles.get(id);
    if (!fileEntry) {
      console.warn(`[app-config-file] Skipping orphaned file '${id}': not in current manifest`);
      failed.push(id);
      continue;
    }

    // Validate mountPath against denylist
    if (isPathDenied(fileEntry.mountPath)) {
      console.warn(`[app-config-file] Skipping file '${id}': mountPath '${fileEntry.mountPath}' is denylisted`);
      failed.push(id);
      continue;
    }

    try {
      const base64Data = await this.backend.fetchSecret(`app-config/file/${id}`);
      const data = Buffer.from(base64Data, 'base64');
      await this.atomicWriteFile(path.resolve(fileEntry.mountPath), data);
      rendered.push(id);
    } catch (err: unknown) {
      console.warn(
        `[app-config-file] Failed to render file '${id}':`,
        err instanceof Error ? err.message : String(err),
      );
      failed.push(id);
    }
  }

  console.log(JSON.stringify({ event: 'files-rendered', count: rendered.length, skipped: failed.length }));
  return { rendered, failed };
}

// Extract from setFile() — shared atomic write
private async atomicWriteFile(absPath: string, data: Buffer): Promise<void> {
  await fs.mkdir(path.dirname(absPath), { recursive: true });
  const tmpPath = `${absPath}.tmp.${process.pid}`;
  const fd = await fs.open(tmpPath, 'w', 0o640);
  try {
    await fd.writeFile(data);
    await fd.datasync();
  } finally {
    await fd.close();
  }
  await fs.rename(tmpPath, absPath);
}
```

Then refactor `setFile()` to call `this.atomicWriteFile()`.

### Step 3: Wire boot-render in bin/control-plane.ts

Add file render call after the existing secret env render block (after line 152):

```typescript
// Boot-time render of files
try {
  const fileRenderResult = await appConfigFileStore.renderAll(readManifest);
  if (fileRenderResult.failed.length > 0) {
    console.warn(`[control-plane] Partial file render: ${fileRenderResult.rendered.length} ok, ${fileRenderResult.failed.length} failed`);
    initResult.warnings.push(`appConfigFile render: ${fileRenderResult.failed.length} files failed to render`);
  } else if (fileRenderResult.rendered.length > 0) {
    console.log(`[control-plane] Rendered ${fileRenderResult.rendered.length} files to mount paths`);
  }
} catch (err) {
  console.error('[control-plane] File render error:', err instanceof Error ? err.message : String(err));
}
```

Import `readManifest` from `app-config.ts` at the top of the file.

## Edge Case Handling

| Edge Case | Handling | Tested |
|-----------|----------|--------|
| Manifest changed since upload | Uses current manifest's mountPath; old path not cleaned up | Unit test |
| mountPath denylisted | Skip with warning log | Unit test |
| mountPath unwritable (EACCES) | Caught by try/catch, added to failed[] | Unit test |
| Blob missing from backend | fetchSecret throws, caught, skip with warning | Unit test |
| File id not in manifest | Orphaned blob, skip with warning | Unit test |
| Store disabled | Return `{ rendered: [], failed: [] }` immediately | Unit test |
| Empty metadata (no files) | Loop body doesn't execute, returns `{ rendered: [], failed: [] }` | Unit test |
