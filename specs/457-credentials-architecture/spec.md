## Problem

15 tests in `packages/orchestrator/src/worker/__tests__/claude-cli-worker.test.ts` fail on develop due to the implement phase's "no file changes" check interacting with test mocks.

The root cause: `packages/orchestrator/src/worker/phase-loop.ts:340-385` defines a `PHASES_REQUIRING_CHANGES` set (line 14) containing `'implement'`. When the implement phase succeeds but `prManager.commitPushAndEnsurePr()` returns `hasChanges: false`, the phase is marked as failed.

Tests that mock `github.getStatus()` default to `has_changes: false` (line 24 of test file). When tests don't override this mock to `true`, the implement phase trips the check and expectations break. The mocking notes at lines 190-193 explicitly acknowledge this.
