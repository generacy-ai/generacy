# Implementation Plan: wizard-env-writer: extract `token` from github-app JSON value

**Feature**: Extract `token` field from JSON-structured `github-app` credential values in `wizard-env-writer.ts`
**Branch**: `592-context-generacy-ai-generacy`
**Status**: Complete

## Summary

The `mapCredentialToEnvEntries` function in `wizard-env-writer.ts` currently treats `github-app` credential values as raw token strings. Cloud-side (generacy-cloud#547) is changing the payload to a JSON object `{ token, installationId, accountLogin, ... }`. This fix splits the `github-app` and `github-pat` branches, adds JSON parsing for `github-app` to extract the `token` field, and fails safe (skips `GH_TOKEN`) when the value is unparseable or missing the token field.

Single-file change + test updates. No new dependencies, no new files, no API changes.

## Implementation Approach

### Step 1: Modify `mapCredentialToEnvEntries` (source)

In `packages/control-plane/src/services/wizard-env-writer.ts`, lines 37-39:

**Before:**
```typescript
if (type === 'github-app' || type === 'github-pat') {
  return [{ key: 'GH_TOKEN', value }];
}
```

**After:**
```typescript
if (type === 'github-app') {
  try {
    const parsed = JSON.parse(value) as { token?: unknown };
    if (typeof parsed.token === 'string' && parsed.token.length > 0) {
      return [{ key: 'GH_TOKEN', value: parsed.token }];
    }
  } catch {
    // unparseable value — skip GH_TOKEN entirely
  }
  return [];
}
if (type === 'github-pat') {
  return [{ key: 'GH_TOKEN', value }];
}
```
