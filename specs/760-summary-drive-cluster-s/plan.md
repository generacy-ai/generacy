# Implementation Plan: Drive cluster GitHub identity from acting account

**Feature**: Thread `gitIdentityLogin` from the github-app credential through `wizard-env-writer.ts` so org-installation clusters use the operator-selected acting account for `GH_USERNAME`/`GH_EMAIL` instead of the installation's `accountLogin`.
**Branch**: `760-summary-drive-cluster-s`
**Date**: 2026-06-05
**Status**: Complete
**Spec**: [spec.md](./spec.md)

## Summary

This is the **consumer half** of a two-repo change. The producer (generacy-cloud#812) seals a user-selected `gitIdentityLogin` field at the top level of the github-app credential JSON during cluster activation. This change teaches `wizard-env-writer.ts` to read that field and prefer it over `accountLogin` when deriving `GH_USERNAME` and `GH_EMAIL`, with fallback to `accountLogin` when the field is missing, empty, or whitespace-only (preserving back-compat for pre-#812 clusters). `identity.ts` gets a comment-only correction: `GH_USERNAME` is the operator-selected acting account, not "the human account the installation belongs to." Resolution order in `resolveClusterIdentity` is unchanged — `CLUSTER_GITHUB_USERNAME` still wins as the explicit escape hatch.

## Technical Context

**Language/Version**: TypeScript (ESM), Node >=22
**Primary Dependencies**: `yaml`, `@generacy-ai/credhelper` (`ClusterLocalBackend`), `node:fs/promises`. No new dependencies.
**Storage**: Reads `.agency/credentials.yaml` (metadata) + `ClusterLocalBackend` (encrypted secret values at `/var/lib/generacy/credentials.dat`). Writes `/var/lib/generacy/wizard-credentials.env` (mode 0600, transient — sourced and deleted by `entrypoint-post-activation.sh`).
**Testing**: `vitest`. Existing suite at `packages/control-plane/__tests__/services/wizard-env-writer.test.ts` (table-driven, real backend, real tmp dirs).
**Target Platform**: Linux container (cluster-base / cluster-microservices), runs as uid 1000 in control-plane process.
**Project Type**: Monorepo packages — `packages/control-plane/` (Node service) and `packages/orchestrator/` (Node service).
**Performance Goals**: N/A — runs once per `bootstrap-complete` lifecycle action.
**Constraints**: Must not regress legacy clusters whose credentials lack `gitIdentityLogin`. Must trim whitespace before length check (Q3/C). Must read field from **top level** of the parsed JSON (Q2/A). Must not change `identity.ts` resolution order (Q4/A).
**Scale/Scope**: One file changed (`wizard-env-writer.ts`), one file's comment updated (`identity.ts`), one test file extended (`wizard-env-writer.test.ts`). ~30 LOC delta plus tests.
