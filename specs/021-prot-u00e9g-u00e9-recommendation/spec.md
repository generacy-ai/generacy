## Purpose

The protégé represents what the human's trained AI predicts they would decide. It applies the human's:
- **Philosophy**: Core values, risk tolerance, time horizon
- **Principles**: Domain-specific decision patterns
- **Patterns**: Observed regularities from past decisions
- **Context**: Current priorities, constraints, energy level

## The Core Question

Traditional AI: "What is the objectively best decision?"
Protégé: "What would THIS human decide if they were as well-informed in as many disciplines as the AI?"

### Recommendation Process
```
1. Load human's knowledge (philosophy, principles, patterns, context)
2. Identify applicable principles for this decision domain
3. Check for context overrides (current priorities, constraints)
4. Apply philosophy (risk tolerance, time horizon)
5. Generate recommendation with reasoning IN TERMS OF human's principles
6. Compare with baseline and explain any differences
```

## Features

### Principle Matching
- Match decision domain to relevant principles
- Handle conflicting principles (use learned weights)
- Apply "unless" exceptions appropriately

### Context Integration
- Check current priorities against decision
- Apply constraint awareness (time, budget)
- Factor in energy level / decision fatigue

### Philosophy Application
- Apply risk tolerance to uncertain decisions
- Consider time horizon (short vs long term)
- Respect absolute boundaries

### Reasoning Articulation
- Generate reasoning in human's "voice"
- Reference specific principles by name
- Explain tradeoffs in terms of human's values

### Baseline Comparison
- Explicit diff with baseline recommendation
- Articulate WHY the difference exists
- Attribute difference to specific principles/context
