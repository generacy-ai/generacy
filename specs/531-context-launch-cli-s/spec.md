# Feature Specification: Launch scaffolder must set DEPLOYMENT_MODE and CLUSTER_VARIANT env vars

**Branch**: `531-context-launch-cli-s` | **Date**: 2026-05-04 | **Status**: Draft | **Issue**: #531

## Summary

The shared cluster scaffolder (`cluster/scaffolder.ts`) generates `docker-compose.yml` files that are missing `DEPLOYMENT_MODE` and `CLUSTER_VARIANT` environment variables. The control-plane service reads these at boot to populate its `/state` endpoint. Without them, control-plane falls back to defaults (`local` / `cluster-base`), which is only correct by coincidence for the cluster-base + local-launch case. This causes incorrect variant reporting for cluster-microservices users and incorrect deployment mode for cloud-deploy users.

## Context

The control-plane reads two env vars at startup:

```ts
const deploymentMode = (process.env['DEPLOYMENT_MODE'] ?? 'local') as DeploymentMode;
const variant = (process.env['CLUSTER_VARIANT'] ?? 'cluster-base') as ClusterVariant;
```

The scaffolder currently writes `GENERACY_CLOUD_URL`, `GENERACY_CLUSTER_ID`, and `GENERACY_PROJECT_ID` but omits `DEPLOYMENT_MODE` and `CLUSTER_VARIANT`.

**Impact**:
- **cluster-microservices + local launch**: control-plane reports `variant: 'cluster-base'` (wrong) — cloud UI shows incorrect variant
- **Cloud-deploy via DigitalOcean**: `DEPLOYMENT_MODE` reads as `local` instead of `cloud` — Claude Max constraint banner missing

## Files

- `packages/generacy/src/cli/commands/cluster/scaffolder.ts` — shared scaffolder that generates `docker-compose.yml` (used by both `launch` and `deploy` commands)

## Fix

Add two env vars to the `environment:` array in the generated docker-compose.yml:

```typescript
environment: [
  `GENERACY_CLOUD_URL=${input.cloudUrl}`,
  `GENERACY_CLUSTER_ID=${input.clusterId}`,
  `GENERACY_PROJECT_ID=${input.projectId}`,
  `DEPLOYMENT_MODE=local`,
  `CLUSTER_VARIANT=${input.variant}`,
],
```

Because both `launch` and `deploy` use the shared scaffolder, the fix lands in one place.

**As a** cloud-deploy user (e.g., DigitalOcean),
**I want** the generated docker-compose.yml to include `DEPLOYMENT_MODE`,
