# Quickstart: Onboarding Slash Command Suite

**Feature**: `382-phase-4-2-cloud` | **Date**: 2026-03-14

## Installation

The onboarding commands are distributed as an agency plugin. Install via the Claude Code marketplace:

```bash
# Install the agency onboard plugin (provides MCP tools)
claude plugin install @generacy-ai/agency-plugin-onboard

# Install the Claude Code command plugin (provides /onboard:* commands)
claude plugin install @generacy-ai/claude-plugin-agency-onboard
```

Or add to your project's `.claude/settings.json`:

```json
{
  "mcpServers": {
    "agency": {
      "command": "npx",
      "args": ["@generacy-ai/agency", "--plugins", "@generacy-ai/agency-plugin-onboard"]
    }
  }
}
```

## Basic Usage

### 1. Assess readiness

```
/onboard:evaluate
```

Checks your project for required files, environment prerequisites, GitHub permissions, and documentation. Outputs a traffic-light report (red/yellow/green) per category.

### 2. Detect and document your tech stack

```
/onboard:stack
```

Scans the project for languages, frameworks, build tools, testing, CI/CD, and infrastructure. Writes results to `.generacy/stack.yaml`.

### 3. Configure plugins

```
/onboard:plugins
```

Presents available Generacy & Agency plugins and recommends selections based on your detected tech stack. Writes to `.generacy/config.yaml`.

### 4. Configure MCP servers

```
/onboard:mcp
```

Recommends and configures MCP servers based on your tech stack. Writes to `.mcp.json`.

### 5. Initialize project configuration

```
/onboard:init
```

Sets up `CLAUDE.md` with project-specific instructions, ensures `.gitignore` has correct entries, and verifies devcontainer configuration.

### 6. Generate architecture documentation

```
/onboard:architecture
```

Analyzes codebase structure and generates an architecture overview document.

### 7. Populate backlog with issues

```
/onboard:backlog
```

Scans for gaps (missing tests, TODOs, tech debt) and creates GitHub issues with your approval.

## Configuration

### Custom MCP server recommendations

Add project-specific MCP server recommendations in `.generacy/mcp-servers.yaml`:

```yaml
servers:
  - name: custom-db-server
    package: "@company/mcp-db"
    command: npx
    args: ["@company/mcp-db"]
    recommend_when:
      - postgresql
      - mysql
    description: "Database query and migration tools"
```

### Stack YAML structure

After running `/onboard:stack`, your `.generacy/stack.yaml` will look like:

```yaml
version: "1"
languages:
  - name: TypeScript
    version: "5.3.2"
    confidence: high
frameworks:
  - name: React
    version: "18.2.0"
    confidence: high
buildTools:
  - name: pnpm
    confidence: high
testing:
  - name: vitest
    confidence: high
packageManager: pnpm
monorepo: true
```

## Troubleshooting

**Commands not found**: Ensure both plugin packages are installed. The MCP server (`agency-plugin-onboard`) provides the tools; the Claude plugin (`claude-plugin-agency-onboard`) provides the commands.

**Stack detection misses a framework**: Detection relies on file-based heuristics (dependency files, config files). If a framework isn't detected, check that its config or dependency entry exists in the project root or workspace root.

**GitHub issues not created**: `/onboard:backlog` requires a GitHub token with `repo` scope. Ensure `GITHUB_TOKEN` is set in your environment.

**`.generacy/stack.yaml` not found by other commands**: Run `/onboard:stack` first. Other commands that consume it (`/onboard:plugins`, `/onboard:mcp`) will report what signals they found.
