# Feature Specification: Wire siblingFanoutHandler and complete agent prompt

**Branch**: `702-follow-up-multi-repo` | **Date**: 2026-05-23 | **Status**: Draft

## Summary

Two gaps prevent multi-repo workflows from functioning end-to-end despite all 6 prerequisite PRs (#693–#700) being merged. The `siblingFanoutHandler` is implemented but never registered as a `phaseAfterHandler`, making the entire feature dead code. Additionally, the agent prompt omits mention of auto-PR behavior, causing the agent to avoid cross-repo edits.

## Background

All 6 multi-repo workflow PRs merged, but the feature is dead code because:
1. **Gap 1 (critical):** `siblingFanoutHandler` is built and tested in `packages/workflow-engine/src/handlers/sibling-fanout.ts` but no caller invokes it. The only `phaseAfterHandlers` entry is a reader that loads `linkedPRs` from a state file that the unregistered writer would have produced. Result: sibling repos are never checked, no branches/PRs created, `linkedPRs` stays empty, `on-sibling-review` gate is vacuously satisfied.
2. **Gap 2:** The agent prompt block in `packages/orchestrator/src/worker/sibling-prompt.ts` doesn't mention auto-PR creation, so the agent may be conservative about cross-repo edits.
