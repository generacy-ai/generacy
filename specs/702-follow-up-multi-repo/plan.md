# Implementation Plan: Wire siblingFanoutHandler and complete agent prompt

**Feature**: Wire the existing `siblingFanoutHandler` into the phase-after hook array and add auto-PR awareness to the agent prompt
**Branch**: `702-follow-up-multi-repo`
**Status**: Complete

## Summary

Two surgical changes wire up the multi-repo workflow feature that is currently dead code:

1. **Register `siblingFanoutHandler`** as the first entry in the `phaseAfterHandlers` array in `claude-cli-worker.ts`. The handler is already implemented and tested in `@generacy-ai/workflow-engine` — it just needs a caller. The existing linkedPRs reader handler must remain second so it can see the state the writer persists.

2. **Append an auto-PR sentence** to `buildSiblingPromptBlock()` in `sibling-prompt.ts` so the agent knows cross-repo edits will be automatically committed and PR'd. Update the snapshot test.

No new files, types, or dependencies are introduced.

## Implementation Approach

### Change 1: Register siblingFanoutHandler (FR-001, FR-002, FR-003, FR-004)

**File**: `packages/orchestrator/src/worker/claude-cli-worker.ts`

At line ~483 where `phaseAfterHandlers` is constructed:

1. Import `siblingFanoutHandler` and `SiblingFanoutContext` from `@generacy-ai/workflow-engine`
2. Create a new handler function that constructs `SiblingFanoutContext` from the `PhaseAfterContext` fields and calls `siblingFanoutHandler()`
3. Place it **before** the existing linkedPRs reader in the array
4. Update the comment on the reader to reflect that the writer is now wired

**Context field mapping** (all already in scope):
| SiblingFanoutContext field | Source in worker scope |
