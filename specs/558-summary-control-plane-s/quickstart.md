# Quickstart — #558 Credential Persistence in Control-Plane

## Prerequisites

- Node >= 22
- pnpm installed
- Master key exists at `/var/lib/generacy/master.key` (auto-created by credhelper-daemon on first boot, or run the dev stack)

## Development Setup

```bash
# Install dependencies
pnpm install

# Build the shared credhelper package first (contains extracted storage modules)
pnpm --filter @generacy-ai/credhelper build

# Build control-plane
pnpm --filter @generacy-ai/control-plane build
```

## Running Tests

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `StorageError: KEY_UNAVAILABLE` | Master key not found | Start credhelper-daemon first, or create key: `openssl rand 32 > /var/lib/generacy/master.key && chmod 600 /var/lib/generacy/master.key` |
| `ENOENT .agency/credentials.yaml` on GET | No credentials written yet | PUT a credential first; GET returns 404 if no metadata exists |
| `EACCES /var/lib/generacy/` | Permission denied | Ensure control-plane runs as uid 1002 or has read/write access |
| 401 on PUT | Missing actor headers | Include `x-generacy-actor-user-id` header (injected by relay in production) |
