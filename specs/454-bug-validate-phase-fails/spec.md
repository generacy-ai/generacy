# Bug Fix: Validate phase fails after worker restart

Workspace package type declarations are not built before validation, causing `pnpm build` (tsc) to fail with TS2307 on freshly bootstrapped workers.

**Branch**: `454-bug-validate-phase-fails` | **Date**: 2026-04-13 | **Status**: Draft

## Summary

The validate phase fails with exit code 2 on workers that have been restarted because `pnpm build` (tsc) cannot resolve workspace package type declarations. The `preValidateCommand` only runs `pnpm install`, which links workspace packages but does not build their `dist/` outputs. After a worker restart (fresh clone), these `dist/` directories are absent, causing TypeScript to fail resolving `@generacy-ai/orchestrator`.

## Symptoms

- `pnpm test && pnpm build` exits with code 2 during validation
- Error: `Cannot find module '@generacy-ai/orchestrator' or its corresponding type declarations` (TS2307)
- Affects **all** validate runs after a worker restart
- Issues #437 and #438 both failed validation due to this

## Root Cause

The validate phase runs `preValidateCommand: 'pnpm install'` followed by `validateCommand: 'pnpm test && pnpm build'`.

`pnpm install` links workspace packages (e.g., `@generacy-ai/orchestrator` → `../../packages/orchestrator`), but does **not** build them. The orchestrator package declares `"types": "./dist/index.d.ts"` in its `package.json`, so `tsc` needs the `dist/` directory to exist.

After a worker restart, `bootstrap-worker.sh` reclones the repo fresh — the `dist/` directories from prior builds are lost. Since `pnpm install` alone doesn't rebuild them, `pnpm build` fails because it can't resolve the workspace dependency types.

Before the restart, `dist/` existed from earlier build phases, so validation succeeded (e.g., issues #430, #431 passed on the same worker).

## Expected Behavior

The validate phase should build workspace dependencies before running the validation command, so it works reliably regardless of worker state.

## Suggested Fix

Change `preValidateCommand` in `WorkerConfigSchema` (at `packages/orchestrator/src/worker/config.ts`) from:

```typescript
preValidateCommand: z.string().default('pnpm install'),
```

to:

```typescript
preValidateCommand: z.string().default('pnpm install && pnpm -r --filter ./packages/* build'),
```

This ensures all workspace packages are compiled before the root `pnpm build` runs type-checking.

## Reproduction

1. Restart a worker container (`docker restart tetrad-development-worker-3`)
2. Trigger validation on any issue with `completed:implementation-review` label
3. Observe exit code 2 from `pnpm build`

Or directly:
```bash
docker exec tetrad-development-worker-3 bash -c "cd /workspaces/generacy && pnpm build"
# → TS2307: Cannot find module '@generacy-ai/orchestrator'
```
