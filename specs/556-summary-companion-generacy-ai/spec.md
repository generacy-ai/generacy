## Summary

Companion to [generacy-ai/cluster-base#20](https://github.com/generacy-ai/cluster-base/issues/20). When the launch scaffolder writes the cluster's `.env` file, add a `GENERACY_BOOTSTRAP_MODE=wizard` line so the entrypoint scripts in cluster-base know to defer repo cloning until after the bootstrap wizard completes.

## Background

cluster-base#20 traced an issue where the orchestrator entrypoint tries to `git clone` the project repo at boot — before the user has any opportunity to provide credentials through the bootstrap wizard. The fix direction (Option C from #20) is to add an explicit mode flag that scripts can branch on:

| `GENERACY_BOOTSTRAP_MODE` | Meaning | Clone timing |
|---|---|---|
| `devcontainer` (default if unset) | User opened cluster via VS Code Dev Containers; credentials pre-configured in `.env` | Clone on boot |
| `wizard` | Cluster booted via `npx generacy launch` or cloud-deploy; credentials arrive post-activation | Defer clone |

This issue covers the launch CLI's writer side — the `wizard` value needs to land in the scaffolded `.env`.

## Scope

Update [scaffoldEnvFile in packages/generacy/src/cli/commands/cluster/scaffolder.ts](https://github.com/generacy-ai/generacy/blob/develop/packages/generacy/src/cli/commands/cluster/scaffolder.ts) (function called from [launch/scaffolder.ts:79-91](https://github.com/generacy-ai/generacy/blob/develop/packages/generacy/src/cli/commands/launch/scaffolder.ts#L79-L91)) to emit one additional line:

```env
# Bootstrap mode — see cluster-base entrypoint scripts
# `wizard` defers repo cloning until credentials arrive via the activation wizard
GENERACY_BOOTSTRAP_MODE=wizard
```

The launch CLI flow always uses wizard mode; no conditional needed.

## Dependencies

- **Predicate:** cluster-base#20 needs to land first (or simultaneously). Otherwise the env var is set but no script reads it — harmless but wasted config.
- **Sibling:** generacy-ai/generacy-cloud#528 — same env var written by the DigitalOcean cloud-deploy path.
- **Coordination point:** the env var name. If cluster-base#20's plan phase settles on a different name (e.g. `GENERACY_LAUNCH_MODE`), this issue's implementation must match.
