# Quickstart: Docker Compose Local Development

## Prerequisites

- Docker Engine 20.10+
- Docker Compose v2.x
- Git (to clone the repo)

Verify installation:
```bash
docker --version    # Docker version 20.10+
docker compose version  # Docker Compose version v2.x
```

## Quick Start

### 1. Clone and Setup

```bash
git clone https://github.com/generacy-ai/generacy.git
cd generacy
```

### 2. Configure Environment

```bash
cp .env.example .env
# Edit .env with your values (API keys, tokens)
```

### 3. Start Services

```bash
./scripts/start-local.sh
```

Or manually:
```bash
docker compose up -d
```

### 4. Verify Services

```bash
# Check all services are running
docker compose ps

# Test orchestrator health
curl http://localhost:3000/health

# Test Redis
docker compose exec redis redis-cli ping
```

### Start/Stop

```bash
# Start all services
./scripts/start-local.sh

```

### Viewing Logs

```bash
# All services
./scripts/logs.sh

# Specific service
./scripts/logs.sh orchestrator
./scripts/logs.sh worker
./scripts/logs.sh redis
```

### Manual Commands

```bash
# Rebuild after dependency changes
docker compose build

# Scale workers
docker compose up -d --scale worker=4

# Execute command in container
docker compose exec orchestrator npm test

# View resource usage
docker compose top
docker stats
```

## Development Workflow

### Hot Reload

Source code is mounted into containers. Changes trigger automatic restart:

1. Edit `services/orchestrator/src/*.ts`
2. Save file
3. Watch logs: `./scripts/logs.sh orchestrator`
4. Service restarts automatically

### Adding Dependencies

```bash
# Add to orchestrator
cd services/orchestrator
npm install new-package

# Rebuild container
docker compose build orchestrator
docker compose up -d orchestrator
```

### Running Tests

```bash
# Run tests in container
docker compose exec orchestrator npm test
docker compose exec worker npm test

# Run with coverage
docker compose exec orchestrator npm run test:coverage
```

### Accessing Redis

```bash
# Redis CLI
docker compose exec redis redis-cli

# Common commands
> KEYS *
> GET session:123
> LRANGE jobs:queue 0 -1
> FLUSHALL  # Clear all data (dev only!)
```

## Troubleshooting

### Services Won't Start

```bash
# Check logs for errors
docker compose logs

# Check if ports are in use
lsof -i :3000
lsof -i :6379

# Rebuild from scratch
docker compose down -v
docker compose build --no-cache
docker compose up -d
```

### Volume Mount Issues

```bash
# Verify mounts
docker compose exec orchestrator ls -la /app/src

# Fix permissions (Linux)
sudo chown -R $USER:$USER services/
```

### Network Issues

```bash
# Inspect network
docker network inspect generacy-network

# Test connectivity
docker compose exec worker ping orchestrator
docker compose exec orchestrator ping redis
```

### Memory/Performance

```bash
# Check resource usage
docker stats

# Increase Docker memory (Docker Desktop)
# Preferences → Resources → Memory → 4GB+
```
