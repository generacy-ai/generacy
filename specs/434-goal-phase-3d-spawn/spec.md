## Summary

## Goal

Phase 3d of the [spawn refactor](https://github.com/generacy-ai/tetrad-development/blob/develop/docs/spawn-refactor-plan.md#phase-3--migrate-orchestrator-spawn-sites). Route the two generic shell spawn sites in cli-spawner through `AgentLauncher` + `GenericSubprocessPlugin`. These run arbitrary `sh -c <cmd>` snippets for validate and pre-validate phases — not Claude-specific.

## Scope

- Migrate [packages/orchestrator/src/worker/cli-spawner.ts:102 `runValidatePhase`](https://github.com/generacy-ai/generacy/blob/develop/packages/orchestrator/src/worker/cli-spawner.ts#L102) to `agentLauncher.launch({ pluginId: "generic-subprocess", intent: { kind: "shell", script }, ... })`.
- Migrate [packages/orchestrator/src/worker/cli-spawner.ts:129 `runPreValidateInstall`](https://github.com/generacy-ai/generacy/blob/develop/packages/orchestrator/src/worker/cli-spawner.ts#L129) similarly.
- Both currently use a `sh -c <cmd>` pattern. `GenericSubprocessPlugin.buildLaunch()` should produce the identical command line.
- Preserve current stdio, env merging, and abort-signal handling.

## Acceptance criteria

- Snapshot tests on composed `sh -c` commands for both validators, byte-identical to pre-refactor baseline.
- Existing validate / pre-validate tests pass unchanged.

## Out of scope

- Changes to validation logic itself.
- Claude-code spawn sites (separate Wave 3 issues).

## Dependencies

- Depends on Wave 1 Agent Launcher issue (`GenericSubprocessPlugin`). Does NOT depend on Wave 2 Claude Plugin issue, so could technically land in Wave 2 if queued earlier — kept in Wave 3 for cohesion with the other `cli-spawner.ts` migrations.
- Parallel-safe with the other Wave 3 issues.
