# Implementation Plan: VS Code tunnel name exceeds 20-char limit

**Feature**: Derive a 20-char tunnel name from cluster UUID to satisfy Microsoft's tunnel name limit
**Branch**: `608-symptoms-even-after-604`
**Status**: Complete

## Summary

`loadOptionsFromEnv()` in `vscode-tunnel-manager.ts` passes the raw `GENERACY_CLUSTER_ID` (36-char UUID) as the tunnel name. Microsoft's tunnel service rejects names longer than 20 characters on first registration. Fix: add `deriveTunnelName(clusterId)` that strips hyphens, prefixes `g-`, and takes the first 18 hex chars (total 20 chars). Update `loadOptionsFromEnv()` to call it. Update existing tests that assert on `tunnelName` values.

## Implementation Details

### Change 1: Add `deriveTunnelName()` (FR-001, FR-003)

**File**: `packages/control-plane/src/services/vscode-tunnel-manager.ts` (new function, before `loadOptionsFromEnv`)

```typescript
export function deriveTunnelName(clusterId: string): string {
  const compact = clusterId.replace(/-/g, '');
  return `g-${compact.slice(0, 18)}`;  // 2 + 18 = 20 chars
}
```

Pure function. Exported for unit testing and potential future reuse.
