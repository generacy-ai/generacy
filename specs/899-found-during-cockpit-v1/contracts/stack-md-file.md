# Contract: per-feature `stack.md` file shape

**Consumer**: humans and agents inspecting a branch's technology context.
**Producer**: upstream `update_agent` MCP tool (post-fix, per [companion-issue.md](./companion-issue.md)).

## Path

`specs/<feature-slug>/stack.md`

- `<feature-slug>` matches the feature branch slug (e.g., `899-found-during-cockpit-v1`).
- Co-located with `spec.md`, `clarifications.md`, `plan.md`, `research.md`, `data-model.md`, `quickstart.md`.

## Ownership

- **Written by**: `/plan` skill, via `update_agent` MCP tool. Nobody else.
- **Overwritten**: on every `/plan` invocation for this branch. Whole-file overwrite (Q4→A).
- **Read by**: humans / agents inspecting the branch. Read-only for all consumers.
- **Hand-editable?**: No. Hand-edits are discarded on next `/plan` run. Durable notes live in `spec.md` or `plan.md`.

## Format

```markdown
# Feature Stack: <feature-slug>

Generated by /plan — do not hand-edit.
Last regenerated: <ISO-8601 date, e.g., 2026-07-10>

## Language / runtime

- **Language**: <extracted from plan.md `**Language/Version**:` field, or "N/A" if absent>
- **Runtime**: <optional; extracted from same line if it names a runtime; else omitted>

## Primary dependencies

- <one bullet per dependency named in plan.md `**Primary Dependencies**:` field>
- <or single line "No dependencies added by this feature" if none>

## Storage

<Prose extracted from plan.md `**Storage**:` field. "N/A" acceptable.>

## Testing

<Prose extracted from plan.md `**Testing**:` field.>

## Project type

<Extracted from plan.md `**Project Type**:` field.>

## Recent activity

- **<ISO date>**: <one-line summary of this /plan invocation, e.g. "Initial plan for issue #899">
```

## Fields

All extracted verbatim from `plan.md` using the same regexes that `update_agent` already uses today (see `extractTechnologies()` in `packages/agency-plugin-spec-kit/src/tools/update-agent.ts`):

- `**Language/Version**:` → **Language / runtime** section
- `**Primary Dependencies**:` → **Primary dependencies** section
- `**Storage**:` → **Storage** section
- `**Testing**:` → **Testing** section
- `**Project Type**:` → **Project type** section

Missing plan.md fields render as `N/A` in the corresponding stack.md section, not omitted (predictable structure).

## Header

The mandatory first three lines:

```markdown
# Feature Stack: <feature-slug>

Generated by /plan — do not hand-edit.
```

- Line 1: H1 with the feature slug.
- Line 2: blank.
- Line 3: fixed generator-owner statement.

This header is the honest contract from Q4→A. Any human or agent opening the file learns immediately that hand-edits will be lost.

## Invariants

1. **Branch ownership**: the path `specs/<feature-slug>/` is exclusive to the feature branch. No other branch writes into it. Merges into base either bring the whole directory in (if base doesn't have it) or delete it (if the epic chooses spec-artifact cleanup) — never a mid-file merge on `stack.md`.
2. **No cross-branch content**: `stack.md` never contains data extracted from another branch. It is a pure function of the current branch's `plan.md`.
3. **Whole-file overwrite**: every `/plan` invocation produces a byte-for-byte-fresh file. No merge, no append, no marker regions, no header-based insertion.
4. **Idempotence under same plan.md**: two consecutive `/plan` invocations against an unchanged `plan.md` produce byte-identical `stack.md` files (except possibly the "Last regenerated" date and "Recent activity" line — these are timestamp fields and are allowed to change). Rationale: predictability under repeated invocation.
5. **No CLAUDE.md coupling**: writing `stack.md` never triggers a read or write of `CLAUDE.md` at the repo root or in the feature directory.

## Non-format

`stack.md` is **not**:

- A place for architecture decisions (that's `research.md`).
- A place for data schemas (that's `data-model.md`).
- A place for API contracts (that's `contracts/`).
- A place for human notes (that's `spec.md` or `plan.md`).
- A summary of the whole feature (that's `spec.md` "Summary").

It is a narrow, generator-owned technology snapshot — the same content `update_agent` used to append to `CLAUDE.md`, moved to a file the branch owns.

## Example

For feature `899-found-during-cockpit-v1` on 2026-07-10:

```markdown
# Feature Stack: 899-found-during-cockpit-v1

Generated by /plan — do not hand-edit.
Last regenerated: 2026-07-10

## Language / runtime

- **Language**: TypeScript 5.x, Node.js ≥22

## Primary dependencies

- vitest — test runner
- node:child_process — git operations in the regression test
- node:fs, node:os, node:path — filesystem primitives for the test

## Storage

N/A. This is a documentation-file rearrangement + regression test. No config, no schema, no state.

## Testing

vitest. One new file: `packages/workflow-engine/src/actions/builtin/speckit/__tests__/managed-file-disjointness.test.ts`.

## Project type

Single-file doc edit (CLAUDE.md) + one new test file. No new package, no new source module.

## Recent activity

- **2026-07-10**: Initial plan for issue #899
```
