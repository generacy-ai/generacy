# Data Model: per-feature `stack.md` file

**Feature Branch**: `899-found-during-cockpit-v1`
**Issue**: [generacy-ai/generacy#899](https://github.com/generacy-ai/generacy/issues/899)

## Overview

This feature does not introduce new runtime entities, schemas, or types in *this* repo. The change is a *file relocation* — technology-info content moves from `<repoRoot>/CLAUDE.md` (shared across all branches) to `<feature_dir>/stack.md` (branch-owned).

The only "data model" is the shape of the per-feature `stack.md` file. The file itself is written by the upstream `update_agent` MCP tool once the companion issue lands; this document captures the target shape so the upstream implementation and this repo's regression test agree on it.

## Entity: per-feature `stack.md`

**Path**: `specs/<feature-slug>/stack.md`

**Ownership**: branch-owned (exclusive to the feature branch whose directory it lives under).

**Lifecycle**:

| Event | Behavior |
|-------|----------|
| First `/plan` invocation on a branch | File is created; whole-file write. |
| Subsequent `/plan` invocations on the same branch | File is overwritten from scratch (Q4→A whole-file overwrite). Prior content is not merged, appended, or preserved. |
| `/tasks`, `/implement`, `/analyze` | Do not touch `stack.md`. Read-only for these phases. |
| Human edits between `/plan` runs | Discarded on next `/plan` invocation. See "Human hand-edits" below. |
| Branch merges into base | File either moves into base (if base doesn't have `specs/<feature-slug>/`) or is deleted after merge cleanup — implementation choice by the epic. Not relevant to this spec. |

**Content shape** (proposed to upstream — see [contracts/stack-md-file.md](./contracts/stack-md-file.md) for the full contract):

```markdown
# Feature Stack: <feature-slug>

Generated by /plan — do not hand-edit.
Last regenerated: <ISO-8601 date>

## Language / runtime

- **Language**: <e.g., TypeScript 5.x>
- **Runtime**: <e.g., Node.js ≥22>

## Primary dependencies

- <package name> — <purpose>

## Storage

<Description or "N/A">

## Testing

<Framework list>

## Project type

<Single-package | multi-package | CLI | web | etc.>

## Recent activity

- **<date>**: <one-line summary of last /plan invocation>
```

The exact field set matches what `update_agent`'s `extractTechnologies()` already parses out of `plan.md`:
- `**Language/Version**:`
- `**Primary Dependencies**:`
- `**Storage**:`
- `**Testing**:`
- `**Project Type**:`

These stay the extractable inputs; only the *destination* changes.

## Non-entity: `CLAUDE.md`

`CLAUDE.md` at the repo root is not a per-feature managed file after this change. It is a *static* file:

- Written by hand.
- Contains project overview, environment setup, and stable notes.
- Contains a pointer to `specs/<feature>/stack.md` for per-feature technology info (added by this PR).
- Never rewritten by `/plan` post-fix.

The existing `## <Feature Name> (#NNN)` sections in `CLAUDE.md` on `develop` are legacy — accumulated over the pre-fix period. Q2→A leave-alone policy: they stay until manual cleanup.

## Human hand-edits

If a human wants to record notes that survive across `/plan` invocations:
- **Durable notes about the branch**: `spec.md` or `plan.md` (both branch-owned, both human-writable end-to-end, both untouched by `/plan` after their initial /plan write).
- **Durable notes about the repo**: `CLAUDE.md` (repo-wide, human-managed after this fix).
- **Not `stack.md`**: it is generator-owned. Hand edits are discarded.

This is deliberately narrow — the "one place for one kind of thing" invariant that Q4→A optimizes for.

## Validation

No schema validation is enforced by this repo. Upstream's `update_agent` is responsible for the write format.

The only *invariant* this repo asserts is the merge-disjointness contract (see [contracts/merge-tree-invariant.md](./contracts/merge-tree-invariant.md)):

> For any two feature branches A and B forked from the same base commit, `git merge-tree <base> <A> <B>` must report zero conflicts, and the diff on either side must not touch `CLAUDE.md`.

That invariant is proved by the presence of the per-feature file (`stack.md`) at a branch-unique path (`specs/<feature-slug>/`), and the absence of any shared-file write.

## Relationships

```
spec.md ───────────┐
                   │  (both human-edited, same branch, no conflict class)
plan.md ───────────┤
                   │
tasks.md ──────────┤  (all under specs/<feature-slug>/, all branch-owned)
                   │
stack.md ──────────┘  (post-upstream-fix — generator-owned, whole-file overwrite)


CLAUDE.md (repo root) ── static, human-managed, contains a one-line pointer
                       to specs/<feature>/stack.md. NOT written by /plan.
```

Every `specs/<feature-slug>/*.md` file is branch-owned; `CLAUDE.md` is repo-owned. The two ownership classes never overlap after this fix — that is the whole point.
