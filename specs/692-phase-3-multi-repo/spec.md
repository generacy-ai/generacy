# Feature Specification: **Phase 3 of [multi-repo workflow support](https://github

**Branch**: `692-phase-3-multi-repo` | **Date**: 2026-05-23 | **Status**: Draft

## Summary

**Phase 3 of [multi-repo workflow support](https://github.com/generacy-ai/tetrad-development/blob/develop/docs/multi-repo-workflows-plan.md).** Coordinates the review phase across the primary PR and its linked siblings.

## Summary

Once Phase 2 lands, a workflow can produce a primary PR plus N linked sibling PRs. The review phase today only knows about the primary — sibling PRs sit as drafts until someone manually flips them to ready-for-review, and the review gate doesn't wait for sibling approvals before considering the workflow merge-ready.

This issue:
1. Mirrors the primary PR's ready-for-review state to all linked sibling PRs when the review phase begins.
2. Adds a new gate condition `on-sibling-review` that pauses the workflow until all linked PRs are approved (label-driven, same mechanism as the existing `on-questions` clarification gate).

## Scope

### Ready-for-review sync

- When the review phase marks the primary PR ready-for-review (in the existing `prManager.markReadyForReview()` or equivalent), iterate `WorkflowState.linkedPRs` and flip each sibling draft to ready-for-review via `gh pr ready`.
- If a sibling has already been marked ready manually, skip — don't error.

### New gate condition

- Extend `GateCondition` in [packages/orchestrator/src/worker/types.ts](packages/orchestrator/src/worker/types.ts) (around line 105) with `on-sibling-review`.
- Implement in [gate-checker.ts](packages/orchestrator/src/worker/gate-checker.ts): the gate is satisfied when every PR in `linkedPRs` has an `approved` label (or equivalent — match whatever label convention the existing review gate uses on the primary).
- Surface waiting state with the same label-driven pause/resume pattern as the clarification gate (see [phase-loop.ts:399-458](packages/orchestrator/src/worker/phase-loop.ts#L399-L458) for the existing pattern).

## Out of scope

- Coordinated **merge** across PRs. Reviewers approve; humans merge. Auto-merge in dependency order is a separate, larger feature.
- Cross-repo CI status aggregation (\"don't approve until all sibling CIs are green\"). Reviewers see CI status on each PR individually.
- Re-syncing PR titles/descriptions when the primary changes — see Issue E out-of-scope.

## Acceptance

- A workflow with linked PRs, when entering review: all linked PRs flip from draft to ready-for-review.
- A workflow configured with the `on-sibling-review` gate pauses until all linked PRs have the approval label, then resumes.
- A workflow with no linked PRs and the `on-sibling-review` gate behaves as a no-op (gate immediately satisfied).

## Dependencies

Hard deps: Issues C (linkedPRs schema) and E (linkedPRs populated).

## Blocks

Nothing; this is the last issue in the planned scope. Follow-on work (cleanup, merge ordering) is deferred.
