# Feature Specification: Migrate orchestrator from WebSocket to SSE for real-time updates

**Branch**: `150-migrate-orchestrator-from-websocket` | **Date**: 2026-01-24 | **Status**: Draft

## Summary

## Summary
Replace the WebSocket implementation in the orchestrator with Server-Sent Events (SSE) for real-time updates. SSE is more efficient and easier to scale.

## Current WebSocket Implementation
Located in `packages/orchestrator/src/websocket/`:
- `handler.ts` - WebSocket connection handler
- `messages.ts` - Message type definitions
- `subscriptions.ts` - Subscription management
- `index.ts` - Exports

## Why SSE over WebSocket
- SSE works over standard HTTP/2 - no special infrastructure needed
- Scales with regular HTTP load balancers (no sticky sessions required)
- Auto-reconnection built into the protocol
- Cloud Run, Lambda, etc. handle it natively
- Simpler to debug (it's just HTTP)
- Our communication is unidirectional (server → client)

## Migration Plan

### 1. Create SSE Module
Create `packages/orchestrator/src/sse/`:
- `stream.ts` - SSE stream creation and management
- `events.ts` - Event type definitions
- `subscriptions.ts` - Subscription management (reuse logic)
- `index.ts` - Exports

### 2. SSE Endpoints
```typescript
// Add to routes
GET /events                    # Global event stream
GET /workflows/:id/events      # Workflow-specific events
GET /queue/events              # Queue events
```

### 3. Event Format
```
event: workflow:progress
id: evt_123
data: {"workflowId": "wf_abc", "step": "verify", "progress": 75}

event: queue:updated
id: evt_124
data: {"action": "added", "item": {...}}
```

### 4. Remove WebSocket
- Delete `packages/orchestrator/src/websocket/` directory
- Remove WebSocket dependencies from package.json
- Update server.ts to remove WebSocket upgrade handling
