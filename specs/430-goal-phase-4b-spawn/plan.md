# Implementation Plan: Phase 4b — Migrate executeCommand / executeShellCommand to AgentLauncher

**Feature**: Route `executeCommand` / `executeShellCommand` through `AgentLauncher` + `GenericSubprocessPlugin` using module-level registration
**Branch**: `430-goal-phase-4b-spawn`
**Status**: Complete

## Summary

Migrate the two public-API spawn utilities in `@generacy-ai/workflow-engine` (`executeCommand` and `executeShellCommand`) from direct `child_process.spawn` to the `AgentLauncher` plugin system. Uses module-level registration to avoid a reverse dependency from `workflow-engine` → `orchestrator`. A fallback path preserves backward compatibility for external npm consumers who never call `registerProcessLauncher()`.

Key implementation elements:
1. Define `LaunchFunction` type and `registerProcessLauncher()` / `getProcessLauncher()` in `workflow-engine`
2. Extend `ProcessFactory.spawn()` options with `detached?: boolean`
3. Refactor `executeCommand` and `executeShellCommand` to use the registered launcher when available, falling back to direct `child_process.spawn`
4. Wire registration at orchestrator boot
5. Add snapshot tests for spawn call composition

## Implementation Approach

### Phase A: Extend ProcessFactory with `detached` support

1. Add `detached?: boolean` to `ProcessFactory.spawn()` options in `worker/types.ts`
2. Update `defaultProcessFactory` (and any other factories) to pass `detached` through to `child_process.spawn`
3. Update `RecordingProcessFactory` to capture `detached` in `SpawnRecord`
4. Add `detached?: boolean` to `LaunchSpec` in `launcher/types.ts`
5. Update `AgentLauncher.launch()` to pass `launchSpec.detached` to the factory
6. Update `GenericSubprocessPlugin.buildLaunch()` to forward `detached` from intent (add `detached` to intent types)

### Phase B: Module-level registration in workflow-engine

1. Create `process-launcher.ts` with:
   - `LaunchFunction` type (matching the spawn signature needed)
   - `registerProcessLauncher(launcher)` — sets module-level variable
   - `getProcessLauncher()` — returns registered launcher or `undefined`
2. Export from `actions/index.ts` and `index.ts`

### Phase C: Refactor executeCommand / executeShellCommand

1. `executeCommand`: when launcher is registered, build a `LaunchRequest` with `kind: 'generic-subprocess'` and delegate spawn to the launcher; preserve all existing logic (StringDecoder, callbacks, timeout, abort, process-group kill)
2. `executeShellCommand`: same approach with `kind: 'shell'`
3. When launcher is not registered, fall back to current direct `child_process.spawn` — identical behavior for external consumers
4. Add `// Wave 5 lint allow-list: direct spawn fallback for external consumers` comment on fallback path

### Phase D: Wire registration at orchestrator boot

1. In the orchestrator's boot/init path, call `registerProcessLauncher()` with a function that creates an `AgentLauncher` launch for the `generic-subprocess` / `shell` intents

### Phase E: Snapshot + regression tests

1. Add `cli-utils-snapshot.test.ts` in workflow-engine: use `RecordingProcessFactory` (or equivalent) to verify spawn call composition for both functions
2. Run existing `cli-utils.test.ts` — all tests must pass unchanged
3. Run existing `cli-spawner-snapshot.test.ts` — must pass unchanged
4. Run full workflow-engine test suite

## Risk Mitigation

| Risk | Mitigation |
|------|-----------|
| Process-group kill breaks after migration | Explicit `detached: true` in LaunchSpec + wrapper-level `process.kill(-pid)` preserved |
| StringDecoder behavior changes | executeCommand keeps its own StringDecoder wrapping around `handle.stdout` — launcher only provides the raw streams |
| External npm consumers break | Fallback to direct spawn when no launcher registered; zero API changes |
| Circular dependency orchestrator ↔ workflow-engine | Module-level registration pattern — workflow-engine defines only local types, no imports from orchestrator |
| `#426` conflict on ProcessFactory | Coordinate: both add optional fields to same interface — merge conflict is trivial |
