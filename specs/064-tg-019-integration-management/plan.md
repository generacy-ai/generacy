# Implementation Plan: TG-019 Integration Management

**Feature**: Integration Management for Generacy VS Code Extension
**Branch**: `064-tg-019-integration-management`
**Status**: In Progress
**Parent Epic**: 044-epic-generacy-vs-code (Phase 9)

## Summary

Implement integration management UI for the Generacy VS Code Extension cloud mode. This includes:
- Tree view showing all integrations and their connection status
- GitHub App connection status with repository details
- Connect/disconnect actions for integrations
- Configuration panel for integration settings
- Webhook configuration UI

## Technical Context

| API | REST via ApiClient | Existing pattern from queue/orgs endpoints |
| State | Polling-based | Same as queue view for real-time updates |

## Key Components

### 1. IntegrationsTreeProvider
- Implements `vscode.TreeDataProvider<IntegrationExplorerItem>`
- Polls `/integrations` endpoint for status updates
- Supports flat and grouped view modes
- Handles authentication state changes

### 2. Tree Items
- `IntegrationTreeItem`: Individual integration entry
- `IntegrationTypeGroupItem`: Category grouping
- `IntegrationEmptyItem`: Placeholder when no integrations
- `IntegrationLoadingItem`: Loading state
- `IntegrationErrorItem`: Error state with retry

### 3. GitHub Integration View
- Shows GitHub App installation status
- Lists connected repositories
- Provides installation/configuration links
- Displays permission scopes

### 4. Configuration Panel
- Quick pick based configuration
- Repository selection
- Webhook management
- Settings persistence

## Implementation Order

1. **Phase 1** (Parallel): API endpoints + Tree items
2. **Phase 2** (Sequential): Tree view provider
3. **Phase 3** (Sequential): GitHub view, status utilities, config panel, webhook UI
4. **Phase 4** (Parallel): Tests + Index/exports
