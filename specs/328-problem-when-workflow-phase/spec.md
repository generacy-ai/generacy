## Summary

## Problem

When a workflow phase fails (e.g., `validate`), the label manager only adds `agent:error` and removes `phase:<current>` + `agent:in-progress`. There is no phase-specific label indicating **which** phase failed.

This was observed on issue #313 where the validate phase failed after the implementation-review gate was satisfied. The issue only got `agent:error` — there was no indication that it was specifically the validate phase that failed without checking worker logs.

## Expected Behavior

When a phase fails, the issue should receive a failure label like `failed:<phase>` (e.g., `failed:validate`) in addition to `agent:error`, so users can see at a glance which phase failed.

## Current Behavior

`LabelManager.onError()` in `packages/orchestrator/src/worker/label-manager.ts:104-118`:
```typescript
async onError(phase: WorkflowPhase): Promise<void> {
    // Only adds agent:error, removes phase:<current> and agent:in-progress
    // No phase-specific failure label
}
```

## Proposed Solution

1. Add `failed:<phase>` labels to `WORKFLOW_LABELS` in `packages/workflow-engine/src/actions/github/label-definitions.ts`
2. Update `LabelManager.onError()` to add `failed:<phase>` alongside `agent:error`
3. Clear `failed:*` labels on `process` events (same as `completed:*` labels are cleared in `LabelMonitorService.processLabelEvent`)

## Additional Context

- Worker logs from issue #313 show: `"Error in phase: removing phase:validate and agent:in-progress, adding agent:error"`
- The validate phase failed with exit code 1 after only 473ms (separate issue: missing `pnpm install`)
