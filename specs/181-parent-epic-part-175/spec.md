## Parent Epic
Part of #175 — Real-time workflow monitoring

## Summary

Notify the user when remote jobs complete, fail, or encounter errors. Currently there is no feedback when a job finishes — the user has to manually check.

## Requirements

### VS Code Notifications
- **Job completed successfully**: Information notification with "View PR" action button (if PR was created)
- **Job failed**: Warning notification with "View Logs" action button
- **Step failed with continueOnError**: Subtle notification (status bar only) — don't interrupt the user
- **Job cancelled**: Information notification

### Notification Format
```
✅ speckit-bugfix completed (31m 22s)
  → PR #62: fix: request_decision options not displaying
  [View PR] [View Details]

❌ speckit-bugfix failed at step "implement" (18m 5s)
  → Error: Task T003 timed out
  [View Logs] [View Details]
```

### Configuration
- `generacy.notifications.enabled` — Master toggle (default: true)
- `generacy.notifications.onComplete` — Notify on success (default: true)
- `generacy.notifications.onError` — Notify on failure (default: true)
- `generacy.notifications.sound` — Play sound on completion (default: false)

### Status Bar Updates
- Show running job count and latest status in status bar
- Clicking opens the job list view
- Animate/flash briefly when a job completes or fails
