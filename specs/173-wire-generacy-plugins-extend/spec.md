# Feature Specification: Wire Generacy plugins to extend Latency base classes

**Branch**: `173-wire-generacy-plugins-extend` | **Date**: 2026-02-08 | **Status**: Draft

## Summary

## Summary

The 6 Generacy plugin packages implement their own standalone plugin classes instead of extending the corresponding Latency abstract base classes. This was the intended architecture from the Latency migration — component plugins should be thin wrappers extending Latency plugins.

## Current State

| Plugin | Should Extend | Missing Dependency |
|--------|--------------|-------------------|
| `generacy-plugin-copilot` | `@generacy-ai/latency-plugin-dev-agent` | Not in package.json |
| `generacy-plugin-claude-code` | `@generacy-ai/latency-plugin-claude-code` | Not in package.json |
| `generacy-plugin-cloud-build` | `@generacy-ai/latency-plugin-ci-cd` | Not in package.json |
| `github-issues` | `@generacy-ai/latency-plugin-github-issues` | Not in package.json |
| `jira` | `@generacy-ai/latency-plugin-jira` | Not in package.json |
| `github-actions` | `@generacy-ai/latency-plugin-github-actions` | Not in package.json |

Additionally, `github-actions/src/plugin.ts` redefines `IssueTracker` locally instead of importing from `@generacy-ai/latency-plugin-issue-tracker`.

## Tasks

For each plugin:
1. Add `@generacy-ai/latency-plugin-*` as a `workspace:*` dependency
2. Update plugin class to extend the corresponding Latency abstract base class
3. Remove locally-redefined interfaces (e.g., `IssueTracker` in github-actions)
4. Ensure facet declarations in manifest align with Latency patterns
5. Tests pass
