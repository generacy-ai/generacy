## Summary

## Goal

Change `publish-preview.yml` from auto-publish-on-push to manual dispatch only, so the `preview` npm channel is not exposed to partial in-progress spawn-refactor phases. Prerequisite for all other spawn-refactor issues.

## Scope

- `.github/workflows/publish-preview.yml`: change trigger from `on: push: branches: [develop]` to `on: workflow_dispatch` (see [current workflow, lines 3-5](https://github.com/generacy-ai/generacy/blob/develop/.github/workflows/publish-preview.yml#L3-L5)).
- Confirm `publish-devcontainer-feature` (invoked via `needs: publish-npm`) is correctly gated by the manual dispatch.
- `release.yml` (stable publishing) stays untouched.
- Add a short note to the workflow file (or a `docs/preview-release-procedure.md`) describing how to cut a preview release manually via `gh workflow run publish-preview.yml --ref develop` or the Actions UI.

## Acceptance criteria

- Pushing a commit to `develop` does NOT trigger preview publishing.
- `gh workflow run publish-preview.yml --ref develop` (or UI "Run workflow") DOES publish preview.
- PR description includes one-line instructions for cutting a preview release.

## Out of scope

- Changes to `release.yml` or stable publishing.
- Any npm version bumps.

- Blocks every other issue in this refactor. Must land first.
