# Implementation Plan: Fix vscode-cli volume mount path in launch CLI scaffolder

**Feature**: Align scaffolder volume mount with cluster-base#34 corrected path
**Branch**: `610-context-cluster-base-34`
**Status**: Complete

## Summary

The launch CLI scaffolder (`packages/generacy/src/cli/commands/cluster/scaffolder.ts`) writes an incorrect Docker named volume mount for VS Code tunnel auth persistence. It currently uses `vscode-cli:/home/node/.vscode-cli` but `code` 1.95.3 writes auth tokens to `/home/node/.vscode/cli/`. This causes tunnel auth to be lost on container recreation.

The fix is a two-line change:
1. Rename the volume from `vscode-cli` to `vscode-cli-state` and update the mount path to `/home/node/.vscode/cli` (line 159)
2. Update the top-level `volumes:` declaration to match (line 252)
