# Quickstart: @generacy-ai/cluster-relay

## Installation

```bash
# From monorepo root
pnpm install

# Build the package
cd packages/cluster-relay
pnpm build
```

## Standalone CLI Usage

### Environment Variables

```bash
# Required
export GENERACY_API_KEY="your-cloud-api-key"

# Optional
export ORCHESTRATOR_URL="http://localhost:3000"    # Default
export ORCHESTRATOR_API_KEY="your-local-api-key"   # For standalone SSE/proxy auth
export RELAY_URL="wss://api.generacy.ai/relay"     # Default
```

### Run

```bash
# Via npx
npx @generacy-ai/cluster-relay

# Via pnpm (development)
cd packages/cluster-relay
pnpm dev

# With CLI flags
npx @generacy-ai/cluster-relay --relay-url wss://staging.generacy.ai/relay --orchestrator-url http://localhost:3000
```

### Graceful Shutdown

Press `Ctrl+C` (SIGINT) or send `SIGTERM`. The relay sends a WebSocket close frame and exits cleanly.

## Library Usage

```typescript
import { ClusterRelay } from '@generacy-ai/cluster-relay';

const relay = new ClusterRelay({
  apiKey: process.env.GENERACY_API_KEY!,
  orchestratorUrl: 'http://localhost:3000',
});

// Register message handler
relay.onMessage((message) => {
  if (message.type === 'api_request') {
    console.log(`Proxying ${message.method} ${message.path}`);
  }
});

// Connect (authenticates and sends handshake automatically)
await relay.connect();

// Push events from orchestrator to cloud
relay.pushEvent('workflows', {
  event: 'workflow:started',
  id: 'evt-123',
  data: { workflowId: 'wf-abc' },
  timestamp: Date.now(),
});

// Override metadata (orchestrator has live data)
relay.setMetadata({
  workerCount: 4,
  activeWorkflows: 2,
});

// Disconnect on shutdown
await relay.disconnect();
```

## Troubleshooting

### "GENERACY_API_KEY is required"
Set the `GENERACY_API_KEY` environment variable or pass `apiKey` in the constructor config.

### Connection keeps reconnecting
- Check that the relay URL is correct and the cloud service is running
- Verify the API key is valid
- Check network connectivity — the relay uses exponential backoff (5s to 300s)
- Look at relay logs for specific error codes

### API proxying returns 502
The relay cannot reach the local orchestrator. Verify:
- The orchestrator is running at the configured `ORCHESTRATOR_URL`
- The port matches (default: 3000)
- Try `curl http://localhost:3000/health` to confirm

### API proxying returns 504
The local orchestrator is taking too long to respond. The default timeout is 30 seconds. Configure `requestTimeoutMs` if needed.

### API proxying returns 401/403
In standalone mode, ensure `ORCHESTRATOR_API_KEY` is set with appropriate scopes (`workflows:read`, `queue:read`).
