# Implementation Plan: CLI worker-count-deriver must read merged cluster config

**Feature**: Fix CLI `reconcileWorkerCount()` to read `cluster.local.yaml` via `readMergedClusterConfig` and stop rewriting `cluster.yaml`.
**Branch**: `712-problem-cli-s`
**Date**: 2026-05-23
**Status**: Complete
**Spec**: [spec.md](./spec.md)
**Issue**: [#712](https://github.com/generacy-ai/generacy/issues/712)

## Summary

`packages/generacy/src/cli/commands/cluster/worker-count-deriver.ts` was added by #708 (PR #710) and not updated when #709 (PR #711) moved the runtime worker-count source of truth to `cluster.local.yaml`. As a result, every `npx generacy up` / `npx generacy update` on a cluster that was scaled via the cloud UI silently resets `WORKER_COUNT` in `.env` back to the `cluster.yaml` template default, destroying the scaled state. Additionally, when the deriver falls back (workers missing, malformed, or clamped), it rewrites the git-tracked `cluster.yaml` — directly violating the goal of #709.

The fix wires the deriver through the existing `readMergedClusterConfig` helper from `@generacy-ai/config` (local-wins per top-level key) and removes the `cluster.yaml` write-back path. The deriver becomes async; the `source` enum gains `'cluster.local.yaml'`; degraded reads on a corrupt local overlay fall back to the canonical layer with a warning. Tests in `worker-count-deriver.test.ts` are updated for async + merged-config paths per the 8-row matrix in `clarifications.md`.

## Complexity Tracking

No constitution violations to justify. The change reduces complexity by:

1. Removing the `cluster.yaml` write-back branch (lines 156–181 of current `worker-count-deriver.ts`).
2. Replacing inline `existsSync` + `readFileSync` + `parseYaml` + manual `RawClusterYamlSchema` with a single call to the centralized `readMergedClusterConfig`.

The only added complexity is the degraded-read fallback (Q1=C), which is a single try/catch wrapping the merged read with a direct canonical-only path on throw. This is justified by the spec's requirement that lifecycle commands stay operable on a corrupted local overlay.
