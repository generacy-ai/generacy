# Implementation Plan: Fix RepoCheckout Dirty-State Blocking Branch Switch

**Feature**: `RepoCheckout.updateRepo()` and `switchBranch()` fail when the working tree has uncommitted changes from a previous worker run
**Branch**: `366-summary-repocheckout`
**Status**: Complete

## Summary

Two private methods in `RepoCheckout` — `updateRepo()` and `switchBranch()` — attempt `git checkout` on a potentially dirty working tree. When tracked files have uncommitted modifications, `git checkout` throws before the `git reset --hard` cleanup is ever reached, dead-lettering the work item. The fix is to discard dirty state with `git reset --hard HEAD` + `git clean -fd` **before** the branch switch in both methods. Additionally, `bootstrap-worker.sh` will pass `--clean` to `generacy setup workspace` so workers always start fresh.

## Implementation Details

### Fix 1: `updateRepo()` — `repo-checkout.ts` lines 185-219

Insert two git commands at the top of the method body, **before** `git fetch origin`:

```typescript
private async updateRepo(checkoutPath: string, branch: string): Promise<void> {
  this.logger.info({ checkoutPath, branch }, 'Updating existing checkout');

  // Discard any leftover dirty state from previous worker runs
  this.logger.debug({ checkoutPath }, 'Discarding dirty state before branch switch');
  await execFileAsync('git', ['reset', '--hard', 'HEAD'], { cwd: checkoutPath });
  await execFileAsync('git', ['clean', '-fd'], { cwd: checkoutPath });

  this.logger.debug({ checkoutPath }, 'Fetching from origin');
  await execFileAsync('git', ['fetch', 'origin'], { cwd: checkoutPath });
  // ... rest unchanged
}
```

### Fix 2: `switchBranch()` — `repo-checkout.ts` lines 102-121

Same pattern — insert before `git fetch origin`:

```typescript
async switchBranch(checkoutPath: string, branch: string): Promise<void> {
  this.logger.info({ checkoutPath, branch }, 'Switching to branch');

  // Discard any leftover dirty state from previous worker runs
  await execFileAsync('git', ['reset', '--hard', 'HEAD'], { cwd: checkoutPath });
  await execFileAsync('git', ['clean', '-fd'], { cwd: checkoutPath });

  await execFileAsync('git', ['fetch', 'origin'], { cwd: checkoutPath });
  // ... rest unchanged
}
```

**Why `-fd` not `-fdx`**: Gitignored files (`node_modules/`, `dist/`) never interfere with `git checkout`. Using `-fdx` would wipe `node_modules` on every checkout, forcing a full reinstall with no benefit (see clarification Q3).

### Fix 3: `bootstrap-worker.sh` — lines 106-112

Change `generacy setup workspace` calls to include `--clean`:

```bash
# Before
generacy setup workspace --config "$CONFIG_PATH"
generacy setup workspace

# After
generacy setup workspace --clean --config "$CONFIG_PATH"
generacy setup workspace --clean
```

The `--clean` flag already exists in `workspace.ts` (line 290) and calls `git reset --hard` before updating. This ensures workers always start fresh when booting, complementing the runtime fix in `repo-checkout.ts`.

### Fix 4: New Tests — `repo-checkout.test.ts`

Add a new `describe` block to the existing test file covering dirty-state recovery for both `updateRepo()` (via `ensureCheckout()`) and `switchBranch()`:

**Test cases for `ensureCheckout()` (updateRepo path)**:
- `reset --hard HEAD` is called before `git fetch origin`
- `clean -fd` is called before `git fetch origin`
- Order: `reset` → `clean` → `fetch` → `checkout` → `reset --hard origin/<branch>`

- `reset --hard HEAD` is called before `git fetch origin`

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Clean flags | `-fd` (not `-fdx`) | Gitignored files don't block checkout; `-fdx` would nuke `node_modules` (clarification Q3-A) |
| `workspace.ts` changes | None (bootstrap flag only) | FR-003 answered as option B: change `bootstrap-worker.sh`, not `cloneOrUpdateRepo()` (clarification Q1-B) |
| New tests | Yes | Bug caused cascading failures across all workers; fix must have regression coverage (clarification Q2-B) |
| Placement of reset+clean | Before `fetch`, not before `checkout` | Cleanest state at start of update sequence; avoids interleaving with fetch artifacts |
