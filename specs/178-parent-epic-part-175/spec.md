## Parent Epic
Part of #175 — Real-time workflow monitoring

## Summary

When the workflow engine spawns `claude -p` processes (in speckit operations: specify, plan, tasks, implement), capture the stdout/stderr output and stream it to the orchestrator so it can be viewed in real-time from the extension.

## Current State

The speckit operations in `packages/workflow-engine/src/actions/builtin/speckit/operations/*.ts` spawn claude via `executeCommand()` in `cli-utils.ts`. The `executeCommand` function collects stdout/stderr into strings but doesn't stream them — it waits for process completion and returns the full output.

The old autodev-worker plugin (`invoker.js`) had an `outputLogger` that captured stdout chunks in real-time and stored them. The current workflow engine doesn't have this capability.

## Requirements

### Streaming stdout Capture in cli-utils
Add an optional streaming callback to `CommandOptions`:

```typescript
export interface CommandOptions {
  // ... existing fields
  /** Callback for stdout chunks (for real-time streaming) */
  onStdout?: (chunk: string) => void;
  /** Callback for stderr chunks */
  onStderr?: (chunk: string) => void;
}
```

Wire these callbacks to the spawn process's stdout/stderr data events in `executeCommand()`.

### Forward Conversation Output as Events
In speckit operations, pass `onStdout`/`onStderr` callbacks that emit events through the executor's event system:

```typescript
const result = await executeCommand('claude', args, {
  // ... existing options
  onStdout: (chunk) => {
    context.emitEvent?.({
      type: 'step:output',
      data: { stream: 'stdout', content: chunk }
    });
  },
});
```

### New Orchestrator Endpoint: `POST /api/jobs/:jobId/logs`
- Append log chunks to a per-job ring buffer (e.g., last 10,000 lines)
- Broadcast to SSE subscribers as `log:append` events

### New Orchestrator Endpoint: `GET /api/jobs/:jobId/logs`
- Return the current log buffer for a job
- Support `?since=<eventId>` for incremental fetching
- Support `?stream=true` for SSE streaming of new log lines

### Log Entry Format
```json
{
  "timestamp": 1234567890,
  "stream": "stdout",
  "stepName": "specify",
  "content": "Reading extension/src/extension.ts...\n"
}
```

## Implementation Notes

- Claude outputs JSON when using `--output-format json`, but also writes tool calls and results interleaved
- Consider parsing claude's JSON output to extract structured tool call information (file reads, edits, bash commands)
- Keep log buffer size bounded to prevent memory issues on long-running jobs
- The `step:output` events from the executor should be separate from the structured workflow events — use a dedicated log channel
