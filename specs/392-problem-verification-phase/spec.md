# Feature Specification: Discovery-Based Workflow Verification

**Branch**: `392-problem-verification-phase` | **Date**: 2026-03-18 | **Status**: Draft

## Summary

Replace hardcoded `pnpm run test` and `pnpm run lint` verification steps in `.generacy/speckit-feature.yaml` and `.generacy/speckit-bugfix.yaml` with the `build.validate` tool, which auto-detects the package manager and discovers all available validation scripts (lint, format:check, typecheck, etc.).

## Problem

The verification phase in both workflow files hardcodes `pnpm` as the package manager and only runs `test` and `lint`. The feature workflow partially mitigates this with `--if-present` and monorepo detection, but the bugfix workflow is fully hardcoded:

```yaml
# bugfix workflow — fully hardcoded
- name: run-tests
  uses: verification.check
  with:
    command: pnpm run test

- name: run-lint
  uses: verification.check
  with:
    command: pnpm run lint
```

This has two problems:
1. **Assumes pnpm** — projects using npm, yarn, or bun will fail or require manual overrides.
2. **Misses validation scripts** — common checks like `format:check`, `typecheck`, `build`, etc. are not run, reducing confidence in the verification phase.

## Proposed Solution

**Depends on:** generacy-ai/agency#323 (`build.validate` tool)

Replace the individual `verification.check` steps with a single `build.validate` step that:
- Auto-detects the project's package manager (pnpm, npm, yarn, bun)
- Discovers available validation scripts from `package.json` (lint, format:check, typecheck, test, etc.)
- Runs all discovered checks, reporting results per-script
- Handles monorepo configurations automatically

## User Stories

### US1: Workflow Author

**As a** workflow author,
**I want** the verification phase to automatically discover and run all validation scripts,
**So that** I don't need to manually enumerate each check and keep it in sync with `package.json`.

**Acceptance Criteria**:
- [ ] Verification phase uses `build.validate` instead of individual hardcoded commands
- [ ] No package manager name is hardcoded in the verification phase

### US2: Project Using Non-pnpm Package Manager

**As a** developer using a project with npm/yarn/bun,
**I want** verification to work without overriding the workflow,
**So that** the speckit workflow is portable across different project setups.

**Acceptance Criteria**:
- [ ] Verification succeeds regardless of which package manager the project uses
- [ ] Package manager is auto-detected, not configured

### US3: Project with Format/Typecheck Scripts

**As a** developer on a project with `format:check` and `typecheck` scripts,
**I want** these to be discovered and executed during verification,
**So that** the verification phase catches formatting and type errors, not just lint and test failures.

**Acceptance Criteria**:
- [ ] All standard validation scripts (lint, format:check, typecheck, test) are discovered and run
- [ ] Missing scripts are gracefully skipped (no errors for scripts not defined in `package.json`)
