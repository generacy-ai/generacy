# Feature Specification: Found during the cockpit v1 integration smoke test (generacy-ai/tetrad-development#88), finding #9

## Summary

Found during the cockpit v1 integration smoke test (generacy-ai/tetrad-development#88), finding #9. With #836 fixed, the watcher now stays alive — but all three P1 issues reached `waiting-for:clarification` BEFORE `/cockpit:watch` was (re)started, and the watch will never mention them: the first poll is a silent baseline and only subsequent transitions emit. A developer who runs queue → watch in the documented order starts a watcher that is silent about everything already waiting on them.

This violates the plan's own coverage principle (docs/epic-cockpit-plan.md: "the sensor emits on every actionable/terminal state including errors — silence is not success"): an issue currently at `waiting-for:*` is actionable regardless of when it got there.

Fix — startup sweep, CLI-side: on the first poll only, emit one NDJSON line per issue currently in an ACTIONABLE state (`waiting-for:*`, `completed:validate`, `failed:*`, `needs:intervention`, `agent:error`), marked `initial: true`; all other states stay silent at baseline. This is NOT rev 2's baseline protocol returning: that emitted every state and required a consumer-side dedupe/seen-set. The rev 3 plugin is stateless per line (print notification + suggestion), so initial lines need no dedupe — and re-surfacing still-pending actionable items on a watch restart is desirable behavior, not noise. Sensor stays a pure sensor; watch.md needs no change beyond treating `initial` lines like any transition.

Alternative considered and rejected: plugin-side one-shot `status --json` sweep at startup — puts actionable-state knowledge (the classifier's job) into command markdown and duplicates it per consumer; the engine should decide, the plugin narrate.

Repro: queue a phase, let issues reach waiting-for:clarification, then start `generacy cockpit watch <epic-ref>` — zero output about the pending gates.

Interim: run /cockpit:status after starting the watch to catch pre-existing actionable states manually.
