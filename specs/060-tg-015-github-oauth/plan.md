## Implementation Summary

### 1. OAuth Flow (auth.ts:204-240)

The login flow uses VS Code's URI handler pattern:
1. Generate CSRF state token using `crypto.getRandomValues`
2. Build authorization URL with redirect to `vscode://generacy-ai.generacy-extension/auth/callback`
3. Open browser via `vscode.env.openExternal`
4. Handle callback via registered URI handler
5. Exchange authorization code for tokens

### 2. Token Management (auth.ts:277-304, 477-565)

- Tokens stored in VS Code SecretStorage (OS keychain)
- Automatic token refresh 5 minutes before expiration
- Graceful handling of expired tokens with user notification

### 3. Authentication State (auth.ts:59-69, 103-117)

Singleton `AuthService` manages:
- Current authentication state
- User information (id, username, email, avatar, tier, organization)
- Event listeners for state changes

### 4. Progressive Tiers (auth.ts:12-19)

Three-tier authentication model:
- **Anonymous**: Can view extension, see features
- **Free**: Full local mode access
- **Organization**: Cloud mode unlocked

### 5. Commands (cloud.ts:21-48)

Registered commands:
- `generacy.login` - Start OAuth flow
- `generacy.logout` - Clear tokens and state
- `generacy.showAccount` - Display account info with actions

## Key Design Decisions

1. **Singleton Pattern**: AuthService uses singleton to ensure consistent state across extension
2. **Event-Driven Updates**: Components subscribe to auth changes via `onDidChange`
3. **VS Code Context**: Sets `generacy.isAuthenticated` for `when` clause menus
4. **Secure Storage**: Uses SecretStorage (not workspace state) for tokens
