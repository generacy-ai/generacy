# Data Model: GitHub OAuth Authentication

## Core Entities

### AuthTier (Enum)

Authentication tiers for progressive access control.

```typescript
enum AuthTier {
  Anonymous = 'anonymous',   // No account - can view extension, see features
  Free = 'free',             // Free account - full local mode access
  Organization = 'organization', // Org member - cloud mode unlocked
}
```

### AuthUser (Interface)

User information from authentication.

```typescript
interface AuthUser {
  id: string;                 // Unique user identifier
  username: string;           // GitHub username
  displayName: string;        // Display name
  email?: string;             // Email address
  avatarUrl?: string;         // Avatar URL
  tier: AuthTier;             // Authentication tier
  organizationId?: string;    // Organization ID (if org member)
  organizationName?: string;  // Organization name (if org member)
}
```

### AuthToken (Interface)

Token information stored securely in SecretStorage.

```typescript
interface AuthToken {
  accessToken: string;        // Access token
  refreshToken?: string;      // Refresh token
  expiresAt?: number;         // Token expiration timestamp (ms)
  scope?: string;             // Token scope
}
```

### AuthState (Interface)

Current authentication state.

```typescript
interface AuthState {
  isAuthenticated: boolean;   // Whether user is authenticated
  user?: AuthUser;            // Current user information
  tier: AuthTier;             // Authentication tier
  lastAuthTime?: number;      // Last authentication time
}
```

### AuthChangeEvent (Interface)

Event emitted when authentication state changes.

```typescript
interface AuthChangeEvent {
  previousState: AuthState;   // Previous state
  newState: AuthState;        // New state
  reason: 'login' | 'logout' | 'token_refresh' | 'token_expired' | 'tier_change';
}
```

## Validation Rules

### AuthUser

- `id`: Non-empty string
- `username`: Non-empty string, valid GitHub username format
- `displayName`: Non-empty string
- `email`: Optional, valid email format if present
- `avatarUrl`: Optional, valid URL if present
- `tier`: Must be valid AuthTier value
- `organizationId`/`organizationName`: Both present or both absent

### AuthToken

- `accessToken`: Non-empty string
- `refreshToken`: Optional but recommended for refresh flow
- `expiresAt`: Optional, unix timestamp in milliseconds if present
