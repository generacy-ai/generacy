# Research: Create Workflow Command

## Technology Decisions

### QuickPick with Preview
**Decision**: Use VS Code's native QuickPick API with live preview panel

**Rationale**:
- Native API provides consistent UX with VS Code
- `onDidChangeActive` event enables real-time preview updates
- `matchOnDescription` and `matchOnDetail` allow template filtering
- Preview in adjacent editor column follows VS Code conventions

**Alternatives Considered**:
1. **WebView Panel**: More customizable but higher complexity and maintenance burden
2. **TreeView Selection**: Less discoverable for single-selection scenarios
3. **Input Box Only**: Insufficient for template preview requirements

### Template Storage
**Decision**: Bundle templates as YAML files in extension resources

**Rationale**:
- Templates are version-controlled with extension code
- No external dependencies or network calls required
- Simple deployment via `extensionUri` path resolution
- Easy to add new templates by dropping YAML files

**Alternatives Considered**:
1. **Embedded in TypeScript**: Harder to maintain multi-line YAML
2. **Remote Template Registry**: Adds network dependency and complexity
3. **User-Defined Templates**: Out of scope for initial implementation

### Name Validation
**Decision**: Regex-based validation with immediate feedback

**Pattern**: `/^[a-zA-Z][a-zA-Z0-9_-]*$/`

**Rationale**:
- Prevents file system issues with special characters
- Ensures valid YAML key names
- Consistent with common naming conventions

## Implementation Patterns

### Singleton Pattern
The `TemplateManager` uses a singleton pattern via `getTemplateManager()`:
```typescript
let templateManagerInstance: TemplateManager | undefined;

export function getTemplateManager(): TemplateManager {
  if (!templateManagerInstance) {
    templateManagerInstance = new TemplateManager();
  }
  return templateManagerInstance;
}
```

**Rationale**: Template content caching is shared across command invocations.

### Error Handling Pattern
Commands use the `withErrorHandling` wrapper:
```typescript
handler: withErrorHandling(createWorkflow, { showOutput: true })
```

**Rationale**: Centralizes error display and telemetry logging.

### File Watcher Pattern
The `WorkflowTreeProvider` uses VS Code's `FileSystemWatcher`:
```typescript
this.fileWatcher = vscode.workspace.createFileSystemWatcher(pattern);
this.fileWatcher.onDidCreate((uri) => this.handleFileChange(uri, 'create'));
```

**Rationale**: Automatic refresh without manual refresh button clicks.
