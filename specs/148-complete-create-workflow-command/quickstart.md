# Quickstart: Create Workflow Command

## Usage

### Creating a New Workflow

1. Open the Command Palette (`Ctrl+Shift+P` / `Cmd+Shift+P`)
2. Type "Generacy: New Workflow"
3. Select a template from the picker (preview appears in side panel)
4. Enter a workflow name
5. The new workflow file opens automatically

### Available Templates

| Template | Best For |
|----------|----------|
| Basic | Simple workflows, getting started |
| Multi-Phase | CI/CD, multi-stage workflows |
| With Triggers | Automated/event-driven workflows |

### Workflow Name Rules

- Must start with a letter
- Can contain: letters, numbers, hyphens (`-`), underscores (`_`)
- Maximum 64 characters
- Must be unique (no duplicates)

## Other Workflow Commands

### Rename Workflow
Right-click a workflow in the explorer or use "Generacy: Rename Workflow"

### Delete Workflow
Right-click a workflow in the explorer or use "Generacy: Delete Workflow"
- Requires confirmation
- Closes file if open in editor

### Duplicate Workflow
Right-click a workflow in the explorer or use "Generacy: Duplicate Workflow"
- Automatically generates unique name (`-copy`, `-copy-2`, etc.)
- Updates workflow name in file content

## Configuration

### Workflow Directory
Default: `.generacy/workflows/`

To change:
```json
{
  "generacy.workflowDirectory": ".workflows/"
}
```

## Troubleshooting

### "No workspace folder open"
- Solution: Open a folder or workspace before creating workflows

### "Workflow already exists"
- Solution: Choose a different name

### Template not loading
- Check the Output panel (View → Output → Generacy)
- Ensure extension is properly installed
