## Summary

Replace the current polling-based queue tree view with a real-time job progress view that shows phase-by-phase, step-by-step execution detail as it happens.

## Current State

The extension has a `QueueTreeView` (`views/cloud/queue/provider.ts`) that polls the orchestrator every 30 seconds. It shows jobs with their status but has no detail about which phase/step is currently executing, no timing information, and no progress bars.

The extension already has local execution UI (`OutputChannel`, `StatusBar`) that handles executor events beautifully — this issue is about bringing that same level of detail to remote/cloud jobs.

## Requirements

### Job List View (Tree View Enhancement)
Enhance the existing queue tree view with:
- Real-time status updates via SSE subscription (`GET /api/events`, from #176)
- Per-job progress indicator (phase X of Y, step X of Y)
- Elapsed time and estimated remaining
- Color-coded status icons (running=blue spinner, completed=green check, failed=red X)
- Group by: status, repository, or flat list (existing grouping modes)

### Job Detail Webview
When clicking a job in the tree view, open a webview panel showing:

```

│ speckit-bugfix: Fix request_decision options  │
│ Status: Running (Phase 5/8)  Elapsed: 18:42  │
```

- Subscribe to job-specific SSE stream (`GET /api/jobs/:jobId/events`)
- Update in real-time as phases/steps start and complete
- Show expandable phase detail with step-level info
- Show error messages inline for failed steps
- Link to PR when pr-creation completes

### Status Bar Integration
- Show currently running job count in status bar
- Click to open the job list view

## Implementation Notes

- The extension already has webview infrastructure (`views/cloud/dashboard/webview.ts`) — follow the same pattern
- Use VS Code's `WebviewPanel` for the detail view
- The SSE client can be a simple `EventSource` wrapper or use `fetch` with streaming
- Consider reusing the local execution `OutputChannel` event handler patterns for consistency
