## Summary

## Goal

Add an end-to-end integration test covering all spawn intent kinds through `AgentLauncher`. Catches regressions in command composition, env inheritance, stdio wiring, and the PTY wrapper that unit and snapshot tests alone can miss.

## Scope

- Add an integration test suite under `tests/integration/spawn/` (or the repo's equivalent) exercising all four intent kinds end-to-end against an argv-inspecting mock `claude` binary:
  - `{ kind: "phase" }` — runs a phase end-to-end with a mocked `/plan`-style command
  - `{ kind: "pr-feedback" }` — runs PR feedback flow with mocked stream-json output
  - `{ kind: "conversation-turn" }` — runs an interactive turn with PTY wrapper, verifies stdin/stdout round-trip
  - `{ kind: "generic-subprocess" }` / `{ kind: "shell" }` — runs a workflow CLI action
- The mock `claude` binary is a small Node or shell script that echoes its argv and env to a fixture file, which the test assertions compare against golden outputs.
- Test failures produce actionable diffs (argv / env / stdout side-by-side).
- Runs in CI under a dedicated job (or folded into existing integration job, whichever the repo conventions prefer).

## Acceptance criteria

- Suite passes green on `develop` after Waves 1-4 land.
- CI runtime overhead is acceptable (target <30s for the full suite).
- Failure output on a deliberately broken branch clearly points at the regressing intent kind and the diff.

## Out of scope

- Running against the real Claude CLI binary (that's an operational smoke test, handled in verification conversations against `tetrad-development`).
- Stress or performance testing.

## Dependencies

- Depends on Waves 1-4 all landing.
- Parallel-safe with the Wave 5 Lint issue.
