# Implementation Plan: Phase 2 Multi-Repo — Cross-Repo Change Fan-Out

**Feature**: Sibling-repo edits get committed, pushed, and turned into linked draft PRs
**Branch**: `691-phase-2-multi-repo`
**Status**: Complete

## Summary

Phase 2 adds the user-visible cross-repo workflow capability: after each workflow phase, a handler detects changes in sibling repositories (populated by Phase 1's `siblingWorkdirs`), commits them to a matching branch, pushes, and opens draft PRs with cross-repo `Closes generacy-ai/<primary-repo>#<issue>` references. Results are persisted to `WorkflowState.linkedPRs` via the idempotent `addLinkedPR` helper from #689.

The handler's internal logic is developed here. Registration against the `phaseAfterHandlers` API depends on #690 (PR #698) — merge order: #690 → #691.

## Implementation Approach

### Component 1: Extend GitStatus (FR-001)

**File**: `packages/workflow-engine/src/actions/github/client/interface.ts:76-82`

Add two fields to `GitStatus`:
- `hasUnpushed: boolean` — true when HEAD is ahead of `origin/<branch>`
- `unpushedCount: number` — count of commits ahead

**File**: `packages/workflow-engine/src/actions/github/client/gh-cli.ts:622-657`

Extend `getStatus()` implementation to run `git rev-list --count origin/<branch>..HEAD`. Handle missing remote tracking branch gracefully (treat as 0 unpushed — new branches haven't been pushed yet, but the working tree dirtiness check already covers them).

### Component 2: Sibling Fan-Out Handler (FR-002 through FR-006)

**File**: `packages/workflow-engine/src/handlers/sibling-fanout.ts` (NEW)

Pure function signature matching the `phaseAfterHandlers` contract from #690:

```typescript
export async function siblingFanoutHandler(ctx: PhaseAfterContext): Promise<void>
```

**Flow per sibling**:

1. **Detect** — `getStatus()` on sibling workdir. Skip if `!has_changes && !hasUnpushed`.
2. **Branch** — `branchExists(primaryBranch)` on sibling. If exists → `checkout`. If not → `getDefaultBranch()` then `createBranch(primaryBranch, defaultBranch)`.
3. **Commit** — `stageAll()` then `commit(primaryLastCommitMessage)`.
4. **Push** — `push('origin', primaryBranch, true)`. Fail loud on error.
5. **PR** — `findPRForBranch(owner, siblingRepo, primaryBranch)`. If null → `createPullRequest()` with draft=true and `Closes generacy-ai/<primaryRepo>#<issueNumber>` body. If exists → skip creation.
6. **Persist** — `addLinkedPR(state, { repo, number, branch, url })` via workflow store.

**Context sourcing** (per clarification Q4):
- Branch name: `getStatus().branch` on primary workdir
- Issue number: from phase-loop context (`item.number`)
- Primary repo name: workspace config path-match (from #687's `resolveSiblingWorkdirs` logic)
- PR title: `gh pr view --json title` on primary at fan-out time

**Error handling** (per clarification Q5):
- Push/PR-create failure → throw (phase fails)
- Detection failure on one sibling → log warning, skip, continue with remaining siblings
- Partial success → leave successful siblings as-is; idempotent retry recovers

### Component 3: Handler Registration (FR-007)

**Integration point**: Once #690 merges, register `siblingFanoutHandler` via the `phaseAfterHandlers` API in the executor or orchestrator bootstrap.

Until #690 lands, the handler is a standalone exported function with its own test suite. Registration is a one-line wiring change.

### Component 4: Short-Circuit Logic (FR-008)

The handler runs after every phase (clarification Q3). It short-circuits immediately when:
- `siblingWorkdirs` is empty (single-repo mode)
- No sibling has changes (all clean)

Cost is one `getStatus()` per sibling per phase — negligible.

## Key Design Decisions

1. **Pure function handler**: No class, no constructor. Takes a context object, returns void or throws. Testable in isolation without executor or orchestrator dependencies.

2. **GitHubClient per sibling**: Create a new `GhCliGitHubClient(siblingWorkdir)` for each sibling's git operations. The existing client is workdir-scoped, so this is the natural pattern.

3. **No rollback on partial failure**: Idempotent retry is the recovery mechanism. Already-pushed siblings no-op on re-run; failed siblings get re-attempted.

4. **Org assumption**: All repos in `generacy-ai` org. Cross-org `Closes` syntax not supported (out of scope).

5. **GitStatus extension is additive**: Adding `hasUnpushed`/`unpushedCount` to the interface is backward-compatible — no existing callers break.
