# Implementation Plan: Publish @stable dist-tag on release to main

**Feature**: Add `@stable` npm dist-tag to `@generacy-ai/generacy` on Changesets release
**Branch**: `656-problem-release-workflow`
**Status**: Complete

## Summary

Add a post-publish step to `.github/workflows/release.yml` that advances the `@stable` npm dist-tag after the Changesets action publishes to npm on merge to `main`. The step extracts the published version from Changesets' `publishedPackages` output, then runs `npm dist-tag add @generacy-ai/generacy@<VERSION> stable`. This preserves the default `@latest` tag and enables the cloud's `launch-claim` endpoint to resolve `packageTag` by project channel.

## Implementation Approach

### Current State (release.yml)

1. Checkout, install, build
2. `changesets/action@v1` — creates release PR or publishes to npm
3. If published, triggers `publish-devcontainer-feature` workflow

### Target State

1. Checkout, install, build
2. `changesets/action@v1` — creates release PR or publishes to npm
3. **NEW: Extract version from `publishedPackages`, run `npm dist-tag add ... stable`**
4. If published, triggers `publish-devcontainer-feature` workflow

### Step Details

The new step is inserted between the Changesets action and the `publish-devcontainer-feature` job trigger. It:

1. **Guards** on `steps.changesets.outputs.published == 'true'` (same condition as the downstream job)
2. **Extracts** the version from `steps.changesets.outputs.publishedPackages` using `jq`:
   ```bash
   VERSION=$(echo '${{ steps.changesets.outputs.publishedPackages }}' \
     | jq -r '.[] | select(.name == "@generacy-ai/generacy") | .version')
   ```
3. **Skips** if the package wasn't in the published set (monorepo may publish other packages)
4. **Runs** `npm dist-tag add @generacy-ai/generacy@$VERSION stable`
5. **Authenticates** via `NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}` (same secret as publish step)

### Key Design Decisions

- **Post-publish tagging** (Option 1) over `--tag stable` publish (Option 2): Keeps `@latest` advancement intact. `npm dist-tag add` is idempotent and atomic.
- **Step in same job** rather than separate job: Avoids extra checkout/setup overhead. The npm auth token and node setup are already configured.
- **jq extraction**: The `publishedPackages` output is a JSON array. Using `jq` is more robust than string parsing.
- **Graceful skip**: If `@generacy-ai/generacy` isn't in `publishedPackages` (e.g., only another package was versioned), the step exits cleanly.
