# Quickstart: Config Discovery for Workspace Setup

## After Implementation

### Standard Usage (auto-discovery)

Mount your project under `/workspaces/` with a `.generacy/config.yaml`, then:

```bash
generacy setup workspace
```

The command scans `/workspaces/*/` for `.generacy/config.yaml` and uses the first (and only) one found.

### Explicit Config Path

```bash
generacy setup workspace --config /workspaces/my-project/.generacy/config.yaml
```

Or via env var:

```bash
CONFIG_PATH=/workspaces/my-project/.generacy/config.yaml generacy setup workspace
```

### CLI Overrides (unchanged)

```bash
# Explicit repo list
generacy setup workspace --repos repo-a,repo-b

# Via env var
REPOS=repo-a,repo-b generacy setup workspace
```

## Error Cases

### No config found

```
ERROR: No .generacy/config.yaml found.
  Provide one via --config, CONFIG_PATH env, or ensure a project
  with .generacy/config.yaml is mounted under /workspaces
```

### Multiple configs found

```
ERROR: Multiple .generacy/config.yaml files found:
  - /workspaces/project-a/.generacy/config.yaml
  - /workspaces/project-b/.generacy/config.yaml
  Use --config or CONFIG_PATH to specify which one.
```

## Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| "No config found" error | No `.generacy/config.yaml` in any `/workspaces/*/` subdir | Add config to your project or use `--config` |
| "Multiple configs found" error | Multiple projects mounted with configs | Use `--config` to pick one |
| Config not detected | Config is nested deeper than one level | Use `--config` with explicit path |
