# Configuration

After initializing your project with `generacy init`, you'll have two key configuration files in your `.generacy/` directory: `config.yaml` for project settings and `generacy.env` for secrets. This page explains what each file does and how to customize them.

## `.generacy/config.yaml`

The `config.yaml` file is the central configuration for your Generacy project. It defines your project identity, repository relationships, workflow defaults, and orchestrator settings.

### Configuration Blocks

The config file has four top-level blocks:

| Block | Required | Purpose |
|-------|----------|---------|
| `project` | Yes | Unique project ID and human-readable name that identify your project in the Generacy platform |
| `repos` | Yes | Defines which repositories belong to this project — the primary repo, optional development repos, and optional clone-only repos |
| `defaults` | No | Sets the default AI agent and base branch for new workflows |
| `orchestrator` | No | Controls polling interval and concurrent worker count for multi-repo orchestration |

### Example Configuration (Single Repo)

Here's a typical single-repo configuration generated by `generacy init`:

```yaml title=".generacy/config.yaml"
schemaVersion: "1"

project:
  # Unique project ID from generacy.ai (format: proj_{alphanumeric})
  id: "proj_myproject456"

  # Human-readable project name
  name: "My Single Repo Project"

repos:
  # Primary repository - the only repo for this project
  # Format: github.com/{owner}/{repo} (no protocol, no .git suffix)
  primary: "github.com/myorg/main-app"

  # No dev or clone repos needed for a single-repo project
  # dev: []    # Optional: other repos to clone for active development
  # clone: []  # Optional: other repos to clone for reference only

defaults:
  # Default agent for workflow execution
  agent: claude-code

  # Default base branch for creating feature branches
  baseBranch: main

# Optional: orchestrator settings can be omitted to use defaults
# orchestrator:
#   pollIntervalMs: 5000  # How often to check for new workflows (min 5000ms)
#   workerCount: 3        # Max concurrent workflow executions (1-20)
```

### Minimal Configuration

Only `project` and `repos` are required. A bare minimum `config.yaml` looks like:

```yaml title=".generacy/config.yaml"
project:
  id: "proj_abc123"
  name: "My Project"

repos:
  primary: "github.com/acme/main-repo"
```

Everything else uses sensible defaults. The `defaults` and `orchestrator` blocks can be added later as needed.

### Key Rules

- **Project ID** — Must match the format `proj_{alphanumeric}` (minimum 12 characters total, e.g., `proj_abc123xyz`). This ID comes from your project on generacy.ai.
- **Repository URLs** — Use the `github.com/{owner}/{repo}` format. No protocol prefix (`https://`) and no `.git` suffix.
- **No duplicate repos** — Each repository can only appear once across `primary`, `dev`, and `clone` lists.
- **Agent names** — Use kebab-case format (e.g., `claude-code`, `cursor-agent`).

:::tip
Run `generacy validate` at any time to check your config for errors. See [Verify Setup](./verify-setup.md) for details.
:::

For the full schema reference including all fields, validation rules, and multi-repo configuration, see the [Configuration Reference](/docs/reference/config/generacy).

## `.generacy/generacy.env`

The `generacy.env` file stores secrets and environment-specific settings. It is **gitignored by default** — never commit this file.

### Set Up Your Environment File

If you haven't already created your env file during [Authentication](./authentication.md) or [Project Setup](./project-setup.md), do so now:

```bash
cp .generacy/generacy.env.template .generacy/generacy.env
```

### Required Variables (Level 1)

For a Level 1 (Agency Only) setup, you need two variables:

```bash title=".generacy/generacy.env"
# GitHub Personal Access Token (PAT) with repo permissions
# Minimum scopes: repo, workflow
# Get one at: https://github.com/settings/tokens/new
GITHUB_TOKEN=ghp_your_token_here

# Anthropic API key for Claude Code agent
# Get one at: https://console.anthropic.com/settings/keys
ANTHROPIC_API_KEY=sk-ant-your_key_here
```

### Auto-Populated Variables

The template also includes variables that are auto-populated from your `config.yaml`:

```bash title=".generacy/generacy.env"
# Your Generacy project ID (auto-populated from config.yaml)
PROJECT_ID=proj_myproject456
```

### Optional Variables

The following variables have defaults and only need to be set if you want to override them:

| Variable | Default | Purpose |
|----------|---------|---------|
| `REDIS_URL` | `redis://redis:6379` | Redis connection for task queue (multi-repo) |
| `LOG_LEVEL` | `info` | Logging level (`debug`, `info`, `warn`, `error`) |
| `BASE_BRANCH` | From `config.yaml` | Override the default base branch |
| `GENERACY_API_URL` | `https://api.generacy.ai` | API endpoint (for self-hosted deployments) |
| `AGENT` | From `config.yaml` | Override the default agent |
| `GITHUB_API_URL` | `https://api.github.com` | GitHub API URL (for GitHub Enterprise) |

:::danger Never commit credentials
The `.generacy/generacy.env` file contains secrets. It is automatically gitignored. **Do not** commit it to version control or share it in public channels. If you accidentally expose a token, revoke it immediately and generate a new one.
:::

For multi-repo orchestration variables like `POLL_INTERVAL_MS` and `WORKER_TIMEOUT_SECONDS`, see [Multi-Repo Setup](./multi-repo.md).
